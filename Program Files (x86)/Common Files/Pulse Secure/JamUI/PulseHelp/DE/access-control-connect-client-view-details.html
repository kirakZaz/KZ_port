Erweiterte Verbindungsdetails anzeigen

Das Dialogfeld „Erweiterte Verbindungsdetails“ zeigt detaillierte Informationen zur ausgewählten Verbindung an.

So können erweiterte Verbindungsdetails angezeigt werden:

  1. Klicken Sie auf eine Verbindung, um sie auszuwählen.
  2. Klicken Sie auf Datei > Verbindungen > Erweiterte Verbindungsdetails.
  3. Die Verbindungsinformationen werden nicht automatisch aktualisiert. So zeigt die verbleibende Sitzungszeit beispielsweise an, wie viel Zeit verbleibt, wenn Sie das Dialogfeld öffnen. Klicken Sie zum Aktualisieren der erweiterten Verbindungsinformationen auf Aktualisieren, oder aktivieren Sie das Kontrollkästchen Automatisch aktualisieren.

Das Fenster „Erweiterte Verbindungsdetails“ enthält die folgenden Informationen:

Feldname

Beschreibung

Verbleibende Sitzungszeit

Die Dauer, während derer die aktuelle VPN-Sitzung aktiv bleibt, bevor die Anmeldeinformationen erneut eingegeben werden müssen oder die Sitzung manuell verlängert werden muss.

Sitzungsdauer

Die Dauer, während derer die aktuelle VPN-Sitzung aktiv ist.

Tunneltyp

Beschreibt, ob die Verbindung einen VPN-Tunnel verwendet oder durch die Port-/Anwendungszuweisung im SAM (Secure Access Manager) erfolgt.

VPN-Typ

Das zum Erstellen des Tunnels verwendete Protokoll (SSL oder ESP).

Zugewiesene IPv4

Die IPv4-Adresse, die dem virtuellen Pulse-Adapter zugewiesen wurde.

Eingehende Byte

Anzahl der Byte, die über den Tunnel empfangen wurden.

Ausgehende Byte

Anzahl der Byte, die über den Tunnel gesendet wurden.

Verbindungsquelle

Beschreibt, wie der Pulse-Client den Verbindungseintrag empfangen hat:

  • Wenn der Wert vorkonfiguriert ist, stammt der Verbindungseintrag von einem Verbindungssatz, der von einem Gateway heruntergeladen wurde.
  • Wenn der Wert vom Benutzer hinzugefügt wurde, dann wurde der Verbindungseintrag manuell vom Endbenutzer über die Pulse-Client-UI hinzugefügt.
  • Wenn der Wert dynamisch ist, so bedeutet dies, dass der Verbindungseintrag durch Starten des Pulse-Clients erzeugt wurde, indem ein Webbrowser eine Verbindung zu einem Pulse Secure-Gateway herstellte und auf der Webseite die Schaltfläche „Start“ drückte.

Verbindungssatz-Name

Dies ist der Name des Verbindungssatzes, der diesen Verbindungseintrag enthält. Ihr Systemadministrator kann anhand dieses Namens den zutreffenden Verbindungssatz auf der Admin-Konsole des entsprechenden Gateways (siehe nächstes Feld) finden, um die Details des Verbindungssatzes anzuzeigen.

Verbindungssatz-Eigentümer

Dies ist der Name des Pulse Secure-Gateways, welches den Verbindungssatz mit diesem Verbindungseintrag zuletzt bearbeitet hat. Dieses Gateway sowie der oben aufgelistete Verbindungssatzname sind die zwei Informationen, die zum Ermitteln der Verbindungssatz-Konfiguration des Clients auf mehreren Pulse Secure-Gateways benötigt werden.

Verbindungssatz Letzte Änderung

Dies ist die Zeit, zu der der Verbindungssatz des Pulse-Clients zuletzt auf dem Gateway aktualisiert wurde.

Verbindungssatz Server-ID

Bei der ersten Implementierung des Pulse-Clients erhält dieser einen Verbindungssatz, der an eine bestimmte Server-ID gebunden ist. Nachdem allen folgenden Verbindungen mit einem Gateway hergestellt wurden, lädt der Client nur dann einen aktualisierten Verbindungssatz herunter, wenn der Verbindungssatz des Gateways eine Server-ID hat, die mit der Server-ID des Verbindungssatzes des Clients übereinstimmt.

Verbindungssatz Download-Host

Dies ist der Server, von dem der Pulse-Client zuletzt seinen Verbindungssatz aktualisiert hat.

Verbindungssatz Download-Zeit

Dies ist die Zeit, zu der der Pulse-Client seinen Verbindungssatz zuletzt aktualisiert hat.

Die obigen Felder werden auch in der Pulse Connect Secure-Administratorkonsole angezeigt.
