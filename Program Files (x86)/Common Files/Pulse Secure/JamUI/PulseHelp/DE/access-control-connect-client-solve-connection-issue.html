Pulse Secure Client-Verbindungsproblem beheben

Sie können die folgenden Fehlerbehebungsinformationen verwenden, um Verbindungsprobleme zu beheben.

Problem

Beschreibung und Lösungsvorschläge

Für die URL einer Verbindung wird „Zufällig ausgewählt“ angezeigt.

Wenn für die Server-URL Zufällig ausgewählt angezeigt wird, wurde diese Verbindung vom Pulse Secure Client-Administrator definiert. Für diese Verbindung wird anstelle eines einzelnen Zielservers eine Liste von Pulse Secure Client-Servern konfiguriert. Pulse Secure Client wählt dann per Zufallsauswahl den Zielserver aus. Für jede Verbindung mit einer Liste von Servern wird die Schaltfläche „Ändern“ angezeigt. Ihr Pulse Secure Client-Administrator kann die Liste so konfigurieren, dass Pulse Secure Client immer eine Verbindung zum ersten Server in der Liste einrichtet, oder zum Server, mit dem Sie zuletzt verbunden waren, oder zu einem nach dem Zufallsprinzip ausgewählten Server.

Pulse Secure Client hat Wireless-Verbindungen deaktiviert und zeigt Deaktiviert – Wired-Verbindung aktiv an.

Der Pulse Secure Client-Administrator kann Pulse Secure Client-Verbindungen konfigurieren, sodass die Wireless-Verbindungen nur aktiv sind, wenn keine Wired-Verbindung zur Verfügung steht. Sobald Sie Ihren Computer mit einer Wired-Verbindung verbinden, beispielsweise durch Anschluss an eine Dockingstation, unterbricht Pulse Secure Client die Wireless-Verbindung und verwendet stattdessen die Wired-Verbindung.

Nach dem Installieren von Pulse Secure Client und dem Deinstallieren von Odyssey Access Client (OAC) kann Ihr System keine Verbindung mehr zu einem Wireless-Netzwerk herstellen.

Pulse Secure Client ersetzt die von OAC bereitgestellten Zugangsdienste. OAC verwaltet jedoch die Wireless-Verbindungen eines Computers, und Pulse Secure Client ist auf die von Windows bereitgestellten Verbindungsfunktionen angewiesen. Nach dem Deinstallieren von OAC müssen Sie die Wireless-Netzwerkeigenschaften von Windows einrichten, um Wireless-Netzwerke zu verwalten.

Während der Anmeldung erscheint folgende Meldung:

Sie sind im Begriff, einen nicht vertrauenswürdigen Server zu authentifizieren. Akzeptieren Sie das Zertifikat für diese Verbindung?

Pulse Secure Client kann die Identität des Servers nicht verifizieren. Dieser Fehler sowie jegliche Zertifikatfehler bedeuten, dass Pulse Secure Client nicht sicherstellen kann, ob Sie eine Verbindung zu einem vertrauenswürdigen Server herstellen. Möglicherweise wurde das Serverzertifikat widerrufen oder ist abgelaufen. Es wurde möglicherweise von einer Zertifizierungsstelle ausgestellt, die von Pulse Secure Client nicht erkannt wird, z. B. wenn Ihre Organisation ein selbstsigniertes Zertifikat verwendet. Wenn Ihr Netzwerkadministrator das entsprechende Recht aktiviert hat, können Sie mit der Verbindung zum Server fortfahren. Sie sollten dies jedoch nur dann tun, wenn Sie von Ihrem Administrator angewiesen wurden, den Zertifikatfehler zu ignorieren.

Nach der Anmeldung erscheint folgende Meldung:

Der Netzwerkzugriff ist aufgrund eines Kompatibilitätsproblems eingeschränkt.

Ihr Endgerät entspricht mindestens einer Anforderung für Ihre Netzwerkzugriffsrichtlinien nicht. Falls die Informationen zur Reparatur von Pulse Secure Client nicht genügend Informationen zur Lösung des Problems enthalten, wenden Sie sich an Ihren Netzwerkadministrator. Nachdem Sie das Problem behoben haben, stellt Pulse Secure Client automatisch eine Verbindung zum Netzwerk her.

Während der Anmeldung erscheint folgende Meldung:

Die Anmeldeinformationen sind ungültig. Versuchen Sie es noch einmal.

Wenn Sie das Kontrollkästchen „Einstellungen speichern“ beim Herstellen einer Pulse Secure Client-Verbindung aktiviert haben, werden die bereitgestellten Anmeldeinformationen bei jeder Herstellung dieser Verbindung ohne Aufforderung zur Eingabe der Anmeldeinformationen verwendet. Pulse Secure Client erkennt jedoch nicht, wenn Sie das Netzwerkkennwort geändert haben – die gespeicherten Einstellungen werden weiter verwendet. Dies kann zum Fehler „Die Anmeldeinformationen sind ungültig“ führen.

Bei der nächsten Anmeldung werden Sie zur Eingabe der Anmeldeinformationen aufgefordert. Wenn Sie zu diesem Zeitpunkt das Kontrollkästchen „Einstellungen speichern“ aktivieren, werden Ihre neuen Anmeldeinformationen gespeichert.

Das Taskleistensymbol und ein Symbol für den Verbindungsbereich zeigen den Status „Fehlgeschlagen“ an:

Verbindungsherstellung fehlgeschlagen

Wenn Sie das Symbol „Verbindungsherstellung fehlgeschlagen“ im Verbindungsbereich sehen, klicken Sie auf die Verbindung, um den Verbindungsstatus anzuzeigen. Der Abschnitt „Details“ enthält den spezifischen Fehler, auf den Sie klicken können, um ein Fenster zu öffnen, das eine detaillierte Beschreibung des Fehlers enthält.

Ihr Anmeldeversuch schlägt fehl, wenn Sie den Zielbereich als Teil Ihres Benutzernamens angeben. Beispiel:

benutzername@bereich

Die Konvention „benutzername@bereich“ wird nicht unterstützt.
