Pulse Secure Client-Benutzeroberfläche

In der Pulse Secure Client-Benutzeroberfläche werden Ihre Pulse Secure Client-Verbindungen aufgeführt. Normalerweise werden Verbindungen zum Pulse Secure Client vom Netzwerkadministrator definiert und implementiert. Ihr Administrator kann Ihnen zudem die Erlaubnis erteilen, neue Verbindungen mit Pulse Secure Client-Servern zu erstellen. Indem Sie eine Verbindung mit dem Webportal eines Pulse Secure Client-Servers aufbauen, können Sie Ihrer Pulse Secure Client-Benutzeroberfläche eine Verbindung mit dem Server hinzufügen.

Pulse Secure Client zeigt darüber hinaus ein Taskleistensymbol an, das den Status der Verbindung anzeigt. Darüber hinaus können Sie über dieses Symbol Verbindungen zu Netzwerken und Sitzungen aufbauen oder trennen und schnell auf die Benutzeroberfläche des Programms zugreifen.

Verbunden mit dem lokalen Netzwerk, aber kein Internetzuggriff verfügbar. An öffentlichen WLAN-Standorten wird häufig ein Captive-Portal bereitgestellt, in dem der Benutzer zunächst Informationen zur Authentifizierung eingeben oder die Servicebedingungen akzeptieren muss, bevor ein Zugriff auf das Netzwerk gewährt wird. Pulse Secure Client erkennt vorhandene Captive-Portale und startet erst dann eine Verbindung mit einem Pulse Secure Client-Server, wenn der Internetzugriff gewährt wurde.

Pulse Secure Client unterstützt den Federal Information Processing Standard (FIPS), welcher sichere Kommunikationspraktiken für die US-Regierungsbehörden definiert. Wenn Ihr Pulse Secure Client-Administrator FIPS aktiviert hat, erscheint am unteren Rand des Pulse Secure Client-Fensters FIPS ein.
