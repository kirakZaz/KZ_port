Verbindung mit einem Netzwerk herstellen

Es muss mindestens eine Verbindung konfiguriert sein, bevor Sie eine Verbindung zu einem Netzwerk herstellen können. Die angezeigten Aufforderungen hängen von der Netzwerkzugriffsumgebung ab.

Zur Verwendung von Pulse Secure Client mit einem Wireless-Netzwerk müssen möglicherweise erst die Wireless-Netzwerkeinstellungen des Endgeräts über Windows oder die auf dem Endgerät installierte Wireless-Gerätesoftware konfiguriert werden. Andernfalls kann Ihr Netzwerkadministrator Wireless-Verbindungen und Scan-Listen definieren und diese in Ihre Pulse Secure Client-Installation integrieren.

Wenn für die Server-URL Zufällig ausgewählt angezeigt wird, bedeutet das, dass diese Verbindung vom Pulse Secure Client-Administrator definiert wurde. Für diese Verbindung wird anstelle eines einzelnen Zielservers eine Liste von Pulse Secure Client-Servern konfiguriert. Pulse Secure Client wählt dann per Zufallsauswahl den Zielserver aus. Für jede Verbindung mit einer Liste von Servern wird die Schaltfläche Ändern angezeigt. Klicken Sie auf die Schaltfläche Ändern, um den Zielserver aus der Liste auszuwählen. Ihr Pulse Secure Client-Administrator kann die Liste so konfigurieren, dass Pulse Secure Client immer eine Verbindung zum ersten Server in der Liste einrichtet, oder zum Server, mit dem Sie zuletzt verbunden waren, oder zu einem nach dem Zufallsprinzip ausgewählten Server.

So verwenden Sie eine definierte Verbindung zum Herstellen einer Netzwerkverbindung:

  1. Klicken Sie für die Verbindung, die Sie herstellen möchten, auf Verbinden.

    Oder

    Klicken Sie in der Taskleiste mit der rechten Maustaste auf das Pulse Secure Client-Symbol, und klicken Sie dann auf Verbinden.

  2. Wenn Sie dazu aufgefordert werden, geben Sie Informationen wie z. B. Benutzername und Kennwort, Bereich, Rolle usw. ein. Welche Eingabeaufforderungen angezeigt werden, hängt von der Einrichtung Ihrer Zugriffsumgebung durch Ihren Netzwerkadministrator ab.

  3. Ein Dialogfeld für den Verbindungsstatus wird angezeigt. Bei erfolgreichem Verbindungsversuch können Sie auf Schließen klicken oder einen Augenblick warten, bis das Dialogfeld automatisch geschlossen wird.

    Falls beim Verbindungsversuch ein Problem aufgetreten ist, werden in einem Reparaturdialogfeld Informationen zu möglichen Maßnahmen zur Behebung oder ein separates Reparaturdialogfeld angezeigt, in dem der Fortschritt der Maßnahmen angezeigt wird, die Pulse Secure Client automatisch zur Behebung des Problems ergriffen hat.

Nach dem Klicken auf die Schaltfläche Verbinden müssen Sie möglicherweise auf folgende Eingabeaufforderungen reagieren:

  • Zertifikat – Falls Pulse Secure Client mit einem Zertifikatserver kommunizieren muss und der Netzwerkadministrator mehr als einen Server konfiguriert hat, werden Sie zur Auswahl eines Servers aufgefordert. Ein von einer Zertifizierungsstelle ausgestelltes Zertifikat prüft, ob die Netzwerkressource, zu der die Verbindung hergestellt werden soll, gültig ist. Falls das Zertifikat von einer vertrauenswürdigen Quelle stammt, wird es automatisch akzeptiert, und die Zertifikataufforderung wird nicht angezeigt. Bei Problemen mit dem Zertifikat werden Sie möglicherweise gefragt, ob Sie das Zertifikat akzeptieren und die Verbindung weiter verwenden möchten.
  • Bereich – Ein Bereich definiert die Netzwerkressourcen, auf die Sie zugreifen können. Falls Sie Zugriff auf mehr als einen Bereich haben, müssen Sie einen auswählen.
  • Anmeldeinformationen – Anhand Ihres Benutzernamens und Kennworts oder des Benutzernamens und Token-Codes übermitteln Sie Ihre Identität an das Zugriffsgerät. Sie werden möglicherweise auch aufgefordert, einen sekundären Benutzernamen und ein Kennwort sowie einen Benutzernamen und ein Kennwort für einen Proxyserver einzugeben. Sie werden möglicherweise regelmäßig von der Authentifizierungsumgebung aufgefordert, das Kennwort und die Token-PIN zu ändern.
  • Rolle – Eine Rolle definiert Sitzungsparameter (Zeitüberschreitungswerte, Anzahl der zulässigen gleichzeitigen Sitzungen usw.), individuelle Einstellungen und aktivierte Zugriffsfunktionen. Ihr Netzwerkadministrator kann das Auswählen der gewünschten Rolle beim Verbinden zulassen.
  • Gleichzeitige Sitzungen – Falls Sie sich mehrmals beim System anmelden, z. B. wenn Sie die gleichen Anmeldeinformationen für die Anmeldung von zwei verschiedenen Computern aus verwenden, werden Sie möglicherweise aufgefordert, die aktuelle Sitzung zu beenden, bevor Sie eine neue beginnen.
  • Hilfsoptionen – Sie werden möglicherweise aufgefordert, Aktionen auszuführen, damit Ihr Endgerät die Sicherheitsrichtlinien erfüllt.

Auf jedem Anmeldebildschirm wird möglicherweise das Kontrollkästchen „Einstellungen speichern“ angezeigt. (Der Administrator kann diese Funktion deaktivieren.) Wenn Sie das Kästchen aktivieren, werden Sie bei der nächsten Anmeldung nicht mehr nach Ihren Daten gefragt. Nachdem Sie die Einstellungen gespeichert haben, können Sie die Funktion „Gespeicherte Einstellungen vergessen“ verwenden, um wieder nach Anmeldeinformationen gefragt zu werden. Beachten Sie, dass die Option „Verbindung unterbrechen“ nicht verfügbar ist, wenn Sie bei einer aktiven Verbindung „Gespeicherte Einstellungen vergessen“ auswählen. Die Funktion „Einstellungen speichern“ ermöglicht das Speichern folgender Informationen:

  • Zertifikatannahme
  • Zertifikatauswahl
  • Bereich
  • Benutzername und Kennwort
  • Proxybenutzername/Kennwort
  • Sekundärer Benutzername/Kennwort
  • Rolle

Die Schritte nach der Beantwortung aller Verbindungsaufforderungen sind von den Zugriffsrichtlinien, die der Netzwerkadministrator konfiguriert hat, und vom Typ des Netzwerkzugriffsgeräts abhängig. Der Verbindungsprozess kann die folgenden Aufgaben umfassen:

  • Softwareaktualisierungen – Pulse Secure Client kann bei bestehender Verbindung automatisch aktualisiert werden. Das System erhält aktualisierte Pulse Secure Client-Software oder Sie erhalten zusätzliche Softwaremodule zur Unterstützung erweiterter Dienste, wie etwa wenn Sie erstmals eine Verbindung zu einem neuen Typ eines Netzwerkzugriffsgeräts herstellen.
  • Host Checker – Host Checker testet Ihr Endgerät, um zu ermitteln, ob es den Sicherheitsrichtlinien des Unternehmens entspricht. Host Checker sucht z. B. nach Antivirussoftware und aktuellen Virensignaturen, dem Status der persönlichen Firewall und nach vorhandener Malware und Spyware. Es kann darüber hinaus die automatische Ausfüllfunktion Ihres Browsers deaktivieren und Ihren Papierkorb leeren. Diese Einstellungen werden von Ihrem Netzwerkadministrator definiert. Wenn Host Checker auf ein Problem stößt, kann eine Meldung angezeigt werden, in der die erforderlichen Schritte zur Lösung des Problems beschrieben sind, oder das Tool kann diese Schritte automatisch ausführen.
  • Cache Cleaner – Cache Cleaner entfernt temporäre Dateien, die von Internet Explorer gespeichert wurden. Cache Cleaner wird als Option von Host Checker ausgeführt.
