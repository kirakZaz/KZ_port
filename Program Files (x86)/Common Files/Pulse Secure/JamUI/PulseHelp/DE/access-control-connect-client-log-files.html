Pulse Secure Client-Protokolldateien

In einer Pulse Secure Client-Protokolldatei werden Informationen erfasst, mit denen Verbindungsprobleme gelöst werden können.

Protokolldateien enthalten Ihre Pulse Secure Client-Verbindungen und deren Eigenschaften. Um die Konfiguration zu speichern, speichern Sie die Protokolle. Ihre aktuelle Pulse Secure Client-Konfiguration wird im Protokollpaket gespeichert. Wenn Sie Pulse Secure Client deinstallieren, haben Sie die Möglichkeit, Ihre Pulse Secure Client-Konfiguration zu speichern, sodass Ihnen bei einer erneuten Installation Ihre Pulse Secure Client-Verbindungen und andere Konfigurationseinstellungen zur Verfügung stehen.

Die Protokollierung erfolgt im Hintergrund. Sie müssen nur dann Änderungen an der Protokollierungsumgebung vornehmen, wenn Sie im Rahmen einer Fehlerbehebungsmaßnahme dazu aufgefordert werden.

Pulse Secure Client schreibt Ereignisse nicht nur in die Pulse Secure Client-Ereignisprotokolle, sondern auch in das Windows-Anwendungsereignisprotokoll (nur Windows Vista und Windows7). Wenn Sie auf Ihrem Computer über Administratorrechte verfügen, können Sie über die Windows-Ereignisanzeige die Pulse Secure Client-Ereignisse anzeigen, die in das Ereignisprotokoll geschrieben wurden. Wenn Sie die Pulse Secure Client-Protokolldateien speichern, beinhalten diese auch Informationen zu den Ereignisprotokollen.

Zur Unterstützung der Fehlerbehebung werden Sie möglicherweise aufgefordert, die folgenden Aufgaben auszuführen:

  • Protokolle mit Anmerkungen versehen – Wenn Sie einer Protokolldatei eine Anmerkung hinzufügen, fügen Sie Text als Markierung an einer bestimmten Position in der Protokolldatei ein. Zur Behebung eines Verbindungsproblems können Sie beispielsweise aufgefordert werden, spezifischen Text als Anmerkung in die Protokolldatei einzufügen, die fehlerhafte Verbindung zu testen und erneut eine Anmerkung in die Protokolldatei einzufügen. Diese Ereignisabfolge ermöglicht einem Supportmitarbeiter das Durchsuchen der Protokolldatei nach dem von Ihnen eingefügten Text. Der Text umschließt die Einträge der Protokolldatei, die das Verbindungsproblem protokollieren.
  • Protokollebene festlegen – Die standardmäßige Protokollebene ist „Normal“. Bei Fehlerhebungsvorgängen werden Sie möglicherweise aufgefordert, die Protokollebene zu „Ausführlich“ zu ändern.
  • Protokolle speichern – Mit „Speichern unter“ werden alle Protokolldateien in einer einzigen ZIP-Datei gesammelt.

Verwandte Themen

  • Protokolldateien mit Anmerkungen versehen
  • Protokollebene festlegen
  • Protokolldateien speichern
  • Protokolldateien hochladen
