Pulse Secure 클라이언트 로그 파일

Pulse Secure 클라이언트 로그 파일 은 연결 문제를 해결하는데 도움이 되는 정보를 추적합니다.

로그 파일에는 Pulse Secure 클라이언트 연결 및 연결 속성이 포함됩니다. 구성을 저장하려면 로그를 저장하면 됩니다. 사용자의 현재 Pulse Secure 클라이언트 구성은 로그 패키지에 저장됩니다. Pulse Secure 클라이언트를 제거하는 경우, Pulse Secure 클라이언트 연결 및 다른 구성 설정을 Pulse Secure 클라이언트 재설치 시 계속 사용할 수 있도록 Pulse Secure 클라이언트 구성을 저장할지 묻는 메시지가 표시됩니다.

로깅은 백그라운드 작업입니다. 문제 해결 과정에서 요청하지 않는 한, 로깅 환경을 변경할 필요는 없습니다.

Pulse Secure 클라이언트 로그 파일 외에도, Pulse Secure 클라이언트는 Windows 응용 프로그램 이벤트 로그에 이벤트를 기록합니다(Windows Vista 및 Windows 7 시스템만 해당). 관리자로 컴퓨터에 액세스하는 경우, Windows 이벤트 뷰어를 사용하여 이벤트 로그에 기록된 Pulse Secure 클라이언트 이벤트를 볼 수 있습니다. Pulse Secure 클라이언트 로그 파일을 저장할 때 이벤트 로그 정보도 포함됩니다.

문제 해결을 지원하기 위해 다음 작업을 수행하도록 요청 받을 수 있습니다.

  • 로그에 주석 달기—로그 파일에 주석을 다는 경우 특정 위치에 로그 파일을 표시하는 텍스트를 삽입합니다. 예를 들어, 연결 문제를 해결하기 위해서 특정 텍스트로 로그 파일에 주석을 단 후, 실패한 연결을 시도하고 다시 로그 파일에 주석을 달도록 요청 받을 수 있습니다. 이 같은 일련의 이벤트로 지원 담당자는 로그 파일에서 사용자가 입력한 텍스트를 검색할 수 있습니다. 이 텍스트는 사용자의 연결 문제를 추적하는 로그 파일 항목을 묶습니다.
  • 로그 수준 설정—기본 로깅 수준은 '기본'입니다. 문제 해결 작업을 위해 사용자는 로깅 수준을 '자세히'로 변경하도록 요청 받을 수 있습니다.
  • 로그 저장—다른 이름으로 저장 작업에서는 모든 로그 파일을 하나의 ZIP 파일로 모을 수 있으며 사용자가 파일의 위치를 지정할 수 있습니다.
