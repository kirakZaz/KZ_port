Pulse Secure 클라이언트 연결 문제 해결

다음 문제 해결 정보를 사용하여 연결 문제를 해결하는 데 도움을 받을 수 있습니다.

문제

설명 및 해결 방법 제안

연결을 위한 URL은 임의로 선택되어 표시됩니다.

서버 URL이 임의로 선택됨으로 표시된다면 이 연결은 Pulse Secure 클라이언트 관리자가 정의한 것입니다. 이 연결은 하나의 대상 서버 대신 Pulse Secure 클라이언트 서버 목록으로 구성되고 Pulse Secure 클라이언트는 대상 서버를 임의로 선택합니다. 변경 버튼은 서버 목록에 포함된 모든 연결의 연결 세부 정보에 나타납니다. Pulse Secure 클라이언트 관리자는 Pulse Secure 클라이언트가 항상 목록에 있는 첫 번째 서버나 가장 최근에 연결한 서버 또는 목록에서 임의로 선택된 서버에 연결되도록 목록을 구성할 수 있습니다.

Pulse Secure 클라이언트가 무선 연결을 비활성화하고 비활성화됨– 유선 연결 활성 상태를 표시합니다.

Pulse Secure 클라이언트 관리자는 유선 연결을 사용할 수 없을 때만 무선 연결이 작동하도록 Pulse Secure 클라이언트 연결을 구성할 수 있습니다. 예를 들어 컴퓨터를 도킹 스테이션에 연결하여 유선으로 연결하면 Pulse Secure 클라이언트가 무선 연결을 해제하고 유선 연결을 사용합니다.

Pulse Secure 클라이언트 설치 및 Odyssey Access Client(OAC) 제거 후, 사용자의 시스템은 더 이상 무선 네트워크에 연결할 수 없습니다.

Pulse Secure 클라이언트는 OAC에서 제공하는 액세스 서비스를 대체합니다. 그러나 OAC는 컴퓨터의 무선 연결을 관리하며 Pulse Secure 클라이언트는Windows에서 제공하는 연결에 의존하게 됩니다. OAC 제거 후에는 무선 네트워크 관리를 위해 반드시 Windows 무선 네트워킹 속성을 설정해야 합니다.

로그인 중에 다음 메시지가 표시됩니다.

신뢰할 수 없는 서버에 인증하려고 합니다. 이 연결에 대한 인증서를 수락하시겠습니까?

Pulse Secure 클라이언트가 서버의 ID를 확인할 수 없습니다. Pulse Secure 클라이언트가 신뢰할 수 있는 서버로 연결 중임을 확인할 수 없을 경우 이 오류 또는 모든 인증서 오류가 발생합니다. 서버 인증서가 취소되었거나 만료되었을 수 있습니다. 조직이 자체 서명된 인증서를 사용하는 경우와 같이 Pulse Secure 클라이언트에서 인식할 수 없는 인증서 기관에서 발행한 인증서일 수 있습니다. 네트워크 관리자가 이 권한을 활성화한 경우 사용자는 계속 진행하여 서버에 연결할 수 있지만, 네트워크 관리자가 인증서 오류를 무시해도 좋다고 통지한 경우에만 그렇게 해야 합니다.

로그인한 후에 다음 메시지가 표시됩니다.

준수 문제로 인한 제한된 네트워크 액세스

사용자의 종점이 네트워크 액세스 정책이 요구하는 요건을 하나 이상 충족하지 못합니다. Pulse Secure 클라이언트 재구성 정보에 이 문제를 해결하기 위한 충분한 정보가 들어 있지 않을 경우 네트워크 관리자에게 문의하십시오. 문제를 해결하면 Pulse Secure 클라이언트는 자동으로 사용자가 네트워크에 연결되도록 합니다.

로그인 중에 다음 메시지가 표시됩니다.

자격 증명이 유효하지 않습니다. 다시 시도해 주십시오.

Pulse Secure 클라이언트 연결이 활성화되었을 때 설정 저장 확인란을 선택한 경우, 연결이 활성화될 때마다 로그인 정보를 입력하라는 메시지 없이 사용자가 제공한 자격 증명이 사용됩니다. 그러나 Pulse Secure 클라이언트는 사용자의 네트워크 암호가 변경된 경우에는 이를 감지할 수 없습니다. 따라서, 계속해서 저장된 설정을 이용하고 결과적으로 “자격 증명이 유효하지 않음” 오류가 발생할 수 있습니다. 이 문제를 해결하려면 파일 > 연결 > 저장된 설정 잊음을 클릭합니다.

다음 번 연결 시 로그인 자격 증명에 관한 메시지가 표시되고, 여기에서 새로운 로그인 정보를 지정할 수 있습니다. 설정 저장 확인란을 선택하면 새로운 자격 증명 정보가 저장됩니다.

시스템 트레이 아이콘과 연결 창 아이콘이 실패 상태로 변경됩니다.

연결 실패

연결 창에서 연결 실패 아이콘이 나타나면 연결을 클릭하여 연결 상태를 표시합니다. 세부정보 섹션에 특정 오류가 표시되면 이를 클릭하여 오류에 관한 자세한 설명을 보여주는 창을 열 수 있습니다.

사용자 이름의 일부로 대상 영역을 지정할 경우 로그인 시도가 실패합니다. 예:

username@realm

username@realm 규칙은 지원되지 않습니다.
