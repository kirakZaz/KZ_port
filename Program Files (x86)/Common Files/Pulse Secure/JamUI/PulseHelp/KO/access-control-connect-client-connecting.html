네트워크에 연결

네트워크에 연결하기 전에 구성된 연결이 최소한 하나 있어야 합니다. 표시되는 메시지는 사용자의 네트워크 액세스 환경에 따라 달라집니다.

무선 네트워크에서 Pulse Secure 클라이언트를 이용하려면, 우선 사용자의 종점에 설치되어 있는 Windows 또는 무선 장치 소프트웨어를 통해 종점의 무선 네트워크 설정을 구성해야 합니다. 또는 네트워크 관리자가 사용자의 무선 연결을 정의하고 목록을 검색하여 이를 사용자의 Pulse Secure 클라이언트 설치에 포함시킬 수 있습니다.

서버 URL이 임의로 선택됨으로 표시된다면 이 연결은 Pulse Secure 클라이언트 관리자가 정의한 것입니다. 이 연결은 하나의 대상 서버 대신 Pulse Secure 클라이언트 서버 목록으로 구성되고 Pulse Secure 클라이언트는 대상 서버를 임의로 선택합니다. 변경 버튼은 서버 목록을 포함하는 모든 연결의 연결 세부정보에 나타납니다. 변경 버튼을 클릭하여 목록에 있는 대상 서버를 선택하십시오. Pulse Secure 클라이언트 관리자는 Pulse Secure 클라이언트가 항상 목록에 있는 첫 번째 서버나 가장 최근에 연결한 서버 또는 목록에서 임의로 선택된 서버에 연결되도록 목록을 구성할 수 있습니다.

정의된 연결을 사용하여 네트워크에 연결하는 방법:

  1. 설정하려는 연결에 대해 연결을 클릭합니다.

    또는

    시스템 트레이에서 Pulse Secure 클라이언트 아이콘을 마우스 오른쪽 버튼으로 클릭하여 팝업 메뉴를 표시한 다음 연결을 클릭한 후 연결을 클릭합니다.

  2. 사용자 이름 및 암호, 영역, 역할 등의 정보를 묻는 메시지에 응답합니다. 표시되는 메시지는 네트워크 관리자가 사용자의 액세스 환경을 구성해 놓은 방식에 따라 달라집니다.

  3. 연결 상태 대화 상자가 표시됩니다. 연결 시도에 성공하면, 닫기를 클릭하거나 자동으로 대화 상자가 닫힐 때까지 잠시 기다립니다.

    연결 시도에 문제가 있다면, 재구성 대화 상자에 문제 해결을 위한 정보가 표시되거나 별도의 재구성 대화 상자가 나타나 Pulse Secure 클라이언트에서 문제 해결을 위해 자동으로 시작한 조치의 진행 정도를 보여줍니다.

연결 버튼을 클릭한 후, 다음 메시지에 응답해야 할 수도 있습니다.

  • 인증서—Pulse Secure 클라이언트가 인증 서버와 통신할 필요가 있고 네트워크 관리자가 둘 이상의 서버를 구성한 경우, 서버를 선택하라는 메시지가 표시됩니다. 인증 기관이 발행한 인증서는 사용자가 연결하려고 하는 네트워크 리소스가 유효한 것임을 확인합니다. 인증서의 출처를 신뢰할 수 있는 경우에는 자동으로 수락되며, 인증서 메시지가 표시되지 않습니다. 인증서와 관련하여 문제가 있는 경우, 인증서를 수락하고 연결을 계속 진행할 것인지를 사용자에게 물을 수 있습니다.
  • 영역—영역은 사용자가 액세스할 수 있는 네트워크 리소스를 정의합니다. 둘 이상의 영역에 액세스할 수 있다면, 이 중 하나를 선택해야 합니다.
  • 자격 증명—사용자 이름 및 암호 또는 사용자 이름 및 토큰 코드로 액세스 장치에 ID를 설정합니다. 프록시 서버의 사용자 이름과 암호와 함께 보조 사용자 이름 및 암호를 묻는 메시지도 나타날 수 있습니다. 사용자의 인증 환경은 정기적으로 사용자에게 암호 또는 토큰 PIN 번호 변경을 요청할 수도 있습니다.
  • 역할—역할 은 세션 매개변수(시간 초과 값, 허용된 동시 세션의 수 등), 개인 설정 및 활성화된 액세스 기능을 정의합니다. 네트워크 관리자는 사용자가 연결 시 원하는 역할을 선택하도록 허용할 수 있습니다.
  • 동시 세션—예를 들어 두 번 이상 시스템에 로그인하였고 두 대의 다른 컴퓨터를 사용하여 로그인할 때 동일한 자격 증명을 사용한 경우, 새로운 세션을 시작하기 전에 현재 세션을 종료하라는 메시지가 표시될 수 있습니다.
  • 재구성 조치—사용자의 종점이 보안 정책을 준수할 수 있도록 작업을 수행하라는 메시지가 표시될 수 있습니다.

설정 저장 확인란이 각 로그인 화면에 표시될 수 있습니다. (관리자가 이 기능을 비활성화 할 수 있습니다.) 확인란을 활성화하면 다음에 로그인할 때 정보를 요청하는 메시지가 표시되지 않습니다. 설정을 저장하면 저장된 설정 잊음 기능을 이용하여 다시 로그인할 때 정보를 요청하는 메시지가 표시되도록 할 수 있습니다. 연결이 활성화된 상태에서 저장된 설정 잊음을 선택하면 연결 일시 중단 옵션을 사용할 수 없습니다. 설정 저장 기능을 사용하면 다음 정보를 저장할 수 있습니다.

  • 인증서 수락
  • 인증서 선택
  • 영역
  • 사용자 이름 및 암호
  • 프록시 사용자 이름/암호
  • 보조 사용자 이름/암호
  • 역할

모든 연결 메시지에 대해 응답한 후에 진행되는 단계는 네트워크 관리자가 구성한 액세스 정책과 네트워크 액세스 장치의 유형에 따라 다릅니다. 연결 과정에는 다음 작업이 포함될 수 있습니다.

  • 소프트웨어 업데이트— Pulse Secure 클라이언트가 연결 시 자동으로 업데이트됩니다. 업데이트된 Pulse Secure 클라이언트 소프트웨어를 시스템에서 수신하거나 새로운 유형의 네트워크 액세스 장치에 처음으로 연결했을 때와 같이 확장된 서비스를 지원하기 위한 추가 소프트웨어 모듈을 수신할 수도 있습니다.
  • 호스트 체커—호스트 체커는 종점이 조직의 보안 정책을 준수하는지 확인하기 위해 종점을 점검합니다. 예를 들어, 호스트 체커는 안티바이러스 소프트웨어와 최신 바이러스 서명, 개인 방화벽 상태 및 맬웨어와 스파이웨어가 있는지 확인할 수 있습니다. 브라우저의 자동 완성 기능 비활성화 및 휴지통 비우기와 같은 작업을 수행할 수도 있습니다. 이러한 설정은 네트워크 관리자가 정의합니다. 호스트 체커가 문제를 발견하면, 문제를 해결하기 위해 사용자가 취해야 할 조치를 메시지로 알려주거나 재구성 작업을 자동으로 시작합니다.
  • Cache Cleaner—Cache Cleaner 는 Internet Explorer에서 저장한 임시 파일을 제거합니다. Cache Cleaner는 호스트 체커의 옵션으로 실행됩니다.
