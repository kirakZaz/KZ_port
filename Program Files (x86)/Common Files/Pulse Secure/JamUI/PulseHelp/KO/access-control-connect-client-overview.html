Pulse Secure 클라이언트 개요

Pulse Secure 클라이언트는 로컬 및 광역 네트워크를 통해 보호된 네트워크 리소스에 대한 인증된 보안 연결을 활성화합니다. 또한 온라인 미팅을 위해 Pulse Secure 클라이언트는 Pulse Collaboration Suite와 통합됩니다. Pulse Secure 클라이언트는 네트워크 관리자가 사용자의 네트워크 액세스 환경을 구성할 수 있도록 하여 네트워크 연결을 단순화합니다.

보안 연결은 네트워크 트래픽을 암호화하여 사용자의 종점 및 종점이 통신하고 있는 서버만이 컨텐츠를 이해할 수 있도록 합니다. 네트워크 관리자는 네트워크 연결을 설정할 때 Pulse Secure 클라이언트가 사용하는 보안 설정을 정의합니다.

인증된 연결은 사용자의 종점과 서버에서 서로에 대해 ID가 입증되었음을 의미합니다. 서버는 자신을 인증하기 위해 신뢰할 수 있는 인증 기관에서 발급한 인증서를 사용하며, 사용자의 종점은 사용자의 로그인 자격 증명을 서버에 제공합니다. 일반적으로는 연결을 위해 사용자에게 사용자 이름과 암호를 요청하지만, 네트워크 관리자는 어떠한 정보도 요청하지 않고 자동으로 인증할 수 있도록 사용자 환경을 구성할 수 있습니다.

보호된 네트워크 리소스는 인증된 연결을 통해 액세스한 모든 네트워크 장치 또는 응용 프로그램입니다. 예를 들어, 조직의 이메일 시스템에 액세스하려면 먼저 네트워크에 로그인합니다. 또한 Pulse Secure 클라이언트로 Pulse Collaboration Suite 클라이언트를 시작하고 미팅에 참여할 수 있습니다. 네트워크 관리자는 영역에서 보호된 네트워크 리소스를 조직화하고, 선택한 영역에 대한 액세스를 부여하여 이같은 리소스에 액세스할 수 있도록 인증합니다. 둘 이상의 영역 으로 액세스하는 경우, Pulse Secure 클라이언트를 통해 네트워크에 연결할 때 원하는 영역을 선택할 수 있습니다. 네트워크 관리자는 호스트 체커 프로그램을 활성화하여 영역에 액세스하는 데 필요한 보안 속성도 정의할 수 있습니다. 호스트 체커는 사용자가 Pulse Secure 클라이언트를 통해 네트워크에 연결할 때 실행됩니다. 호스트 체커는 사용자의 종점을 점검하여 현재 바이러스 및 맬웨어 보호와 같은 필수 소프트웨어를 가지고 있는지 확인합니다 또한 관리자는 특정 네트워크 기반 애플리케이션에만 액세스를 허용하도록 Pulse Secure 클라이언트 연결을 구성할 수 있습니다.

네트워크 관리자는 사용자가 네트워크에 연결된 후 연결이 이동성을 가질 수 있도록 액세스 정책을 구성할 수 있습니다. 예를 들어, 집에서 회사 네트워크로 노트북을 연결한 경우, 연결 해제할 필요 없이 노트북을 사무실로 가져 올 때 재연결하면 됩니다. 집에서는 노트북을 닫기만 하면 됩니다. 사무실에 도착해서 노트북을 열면 Pulse Secure 클라이언트가 변경된 네트워크 환경을 감지합니다. Pulse Secure 클라이언트는 사용자의 연결 프로필 정보를 사용하여 적절한 액세스를 위해 연결을 구성하고, 새로운 환경에서 사용자를 인증합니다. 관리자가 Pulse Secure 클라이언트의 위치 인식 기능을 구성한 경우, 사용자는 시작 시 적절한 네트워크로 연결됩니다. 예를 들어, 사용자의 종점이 원격 위치에서 시작되었다면 Pulse Secure 클라이언트는 자동으로 WAN 액세스 장치로 연결됩니다.

Pulse Secure 클라이언트를 사용하면 네크워크 관리자는 사용자가 네트워크에 연결할 때 사용자의 종점에 새로운 소프트웨어와 중요한 소프트웨어 업데이트를 “권유(push)”할 수 있습니다.
