Connessione a una rete

Per poter stabilire la connessione a una rete, è necessario che sia configurata almeno una connessione. Le richieste visualizzate dipendono dall'ambiente di accesso alla rete.

Per utilizzare Pulse Secure Client con una rete wireless, potrebbe essere necessario configurare prima le impostazioni di rete wireless dell'endpoint tramite Windows o il software per dispositivi wireless installato nell'endpoint. In alternativa, l'amministratore di rete può definire le connessioni wireless e gli elenchi di ricerca e includerli nell'installazione di Pulse Secure Client dell'utente.

Se l'URL del server viene visualizzato come Selezionato a caso, significa che la connessione è stata definita dall'amministratore di Pulse Secure Client. Invece che con un server di destinazione, la connessione viene configurata con un elenco di server Pulse Secure Client selezionati in maniera casuale. Un pulsante Modifica viene visualizzato nei dettagli della connessione per ogni connessione che include un elenco di server. Fare clic sul pulsante Modifica per selezionare il server di destinazione dall'elenco. L'amministratore di Pulse Secure Client può configurare l'elenco in modo che Pulse Secure Client si connetta sempre al primo server dell'elenco, al server a cui si è effettuato l'accesso più di recente o a un server scelto casualmente dall'elenco.

Per utilizzare una connessione definita per connettersi a una rete:

  1. Fare clic su Connetti per la connessione che si desidera stabilire.

    Oppure

    Nell'area di notifica, fare clic con il pulsante destro del mouse sull'icona di Pulse Secure Client per visualizzare il menu a comparsa, scegliere la connessione, quindi fare clic su Connetti.

  2. Rispondere alle richieste di informazioni, fornendo ad esempio nome utente e password, area di autenticazione, ruolo e così via. Le richieste visualizzate dipendono dal modo in cui l'amministratore di rete ha configurato l'ambiente di accesso.

  3. Viene visualizzata una finestra di dialogo con lo stato della connessione. Se il tentativo di connessione ha avuto esito positivo, è possibile fare clic su Chiudi o attendere che la finestra si chiuda automaticamente dopo pochi secondi.

    Se si verifica un problema durante il tentativo di connessione, in una finestra di dialogo di correzione vengono visualizzate informazioni sulle possibili azioni da intraprendere per la sua risoluzione oppure ne viene visualizzata un'altra separata in cui è mostrato lo stato di avanzamento delle misure avviate automaticamente da Pulse Secure Client per risolvere il problema.

Dopo aver fatto clic sul pulsante Connetti, potrebbe essere necessario rispondere alle seguenti richieste:

  • Certificato - Se Pulse Secure Client deve comunicare con un server per certificati e l'amministratore di rete ha configurato più server, viene richiesto di scegliere un server. Un certificato emesso da un'autorità di certificazione verifica la validità della risorsa di rete a cui si sta eseguendo la connessione. Se il certificato proviene da una fonte attendibile, viene automaticamente accettato e non viene visualizzata alcuna richiesta di certificato. In caso di problema con un certificato, potrebbe essere richiesto se accettare il certificato e procedere con la connessione.
  • Area di autenticazione - Un'area di autenticazione definisce le risorse di rete a cui è possibile accedere. Se si dispone di accesso a più aree di autenticazione, è necessario selezionarne una.
  • Credenziali - Il nome utente e la password o il nome utente e il codice token definiscono l'identità dell'utente per il dispositivo di accesso. Potrebbe anche essere richiesto di immettere nome utente e password secondari o nome utente e password per un server proxy. L'ambiente di autenticazione potrebbe richiedere periodicamente di cambiare la password o il numero PIN del token.
  • Ruolo - Il ruolo definisce i parametri relativi alle sessioni (valori di timeout, numero di sessioni simultanee consentite e così via), le impostazioni personalizzate e le funzionalità di accesso attivate. L'amministratore di rete può consentire all'utente di scegliere il ruolo desiderato quando si connette.
  • Sessioni simultanee - Se si accede al sistema più volte, ad esempio, e si utilizzano le stesse credenziali per l'accesso da due computer diversi, potrebbe essere richiesto di terminare la sessione corrente prima di iniziarne una nuova.
  • Azioni di correzione - Potrebbe essere richiesta l'esecuzione di alcune azioni affinché l'endpoint si conformi ai criteri di sicurezza.

È possibile che venga visualizzata una casella di controllo Salva impostazioni su ciascuna schermata di accesso. L'amministratore può disattivare questa funzionalità. Se la casella di controllo viene selezionata, al successivo accesso non viene richiesto di immettere le informazioni. Se le impostazioni vengono salvate, è possibile utilizzare la funzionalità Dimentica impostazioni salvate per ripristinare la richiesta delle informazioni di accesso. L'opzione Sospendi non è disponibile se si sceglie Dimentica impostazioni salvate in presenza di una connessione attiva. La funzione Salva impostazioni consente di salvare le seguenti informazioni:

  • Accettazione del certificato
  • Selezione del certificato
  • Area di autenticazione
  • Nome utente e password
  • Nome utente proxy e relativa password
  • Nome utente secondario e relativa password
  • Ruolo

I passaggi che seguono dopo aver risposto a tutte le richieste relative alla connessione dipendono dai criteri di accesso configurati dall'amministratore di rete e dal tipo di dispositivo di accesso alla rete. Il processo di connessione può includere le seguenti attività:

  • Aggiornamenti software - Pulse Secure Client può essere aggiornato automaticamente in fase di connessione. È possibile che il sistema riceva il software Pulse Secure Client aggiornato o che l'utente riceva moduli software aggiuntivi a supporto di maggiori servizi, ad esempio quando si connette a un nuovo tipo di dispositivo di accesso alla rete per la prima volta.
  • Host Checker - Host Checker esamina l'endpoint per verificare se soddisfa i criteri di sicurezza dell'organizzazione. Ad esempio, potrebbe verificare la disponibilità di software antivirus e relative firme aggiornate, lo stato del firewall personale e l'eventuale presenza di malware e spyware. Può inoltre eseguire attività quali la disabilitazione della funzione completamento automatico del browser e lo svuotamento del Cestino. Queste impostazioni vengono definite dall'amministratore di rete. Se Host Checker rileva un problema, può visualizzare un messaggio in cui vengono descritti i passaggi da intraprendere per la risoluzione oppure avviare automaticamente le operazioni di correzione.
  • Cache Cleaner rimuove i file temporanei memorizzati da Internet Explorer. Cache Cleaner viene eseguito come opzione di Host Checker.
