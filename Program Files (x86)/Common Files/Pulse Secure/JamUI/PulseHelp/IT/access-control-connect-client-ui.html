Interfaccia di Pulse Secure Client

Nell'interfaccia di Pulse Secure Client sono elencate le connessioni dello stesso. In genere, è l'amministratore di rete che definisce e distribuisce le connessioni a Pulse Secure Client. L'amministratore può anche consentire all'utente la creazione di nuove connessioni ai server Pulse Secure Client. La connessione di un server Pulse Secure Client al portale Web può comportare l'aggiunta di una connessione a tale server nell'interfaccia dello stesso.

Pulse Secure Client visualizza inoltre nell'area di notifica un'icona che segnala lo stato di connessione, consente di connettersi e disconnettersi da reti e riunioni e di accedere rapidamente all'interfaccia del programma.

Connessione sospesa

Connesso alla rete locale ma nessun accesso a Internet disponibile. Le postazioni Wi-Fi pubbliche spesso implementano un captive portal che richiede l'immissione delle informazioni di autenticazione o l'accettazione dei termini di servizio da parte dell'utente affinché l'accesso sia consentito. Pulse Secure Client rileva la presenza di captive portal e non avvia la connessione a un server Pulse Secure Client finché l'accesso non viene consentito.

Pulse Secure Client supporta il Federal Information Processing Standard (FIPS), che definisce i requisiti di comunicazione sicura per il governo statunitense. Se l'amministratore di Pulse Secure Client ha attivato il FIPS, il messaggio FIPS attivato viene visualizzato in prossimità della parte inferiore della finestra di Pulse Secure Client.
