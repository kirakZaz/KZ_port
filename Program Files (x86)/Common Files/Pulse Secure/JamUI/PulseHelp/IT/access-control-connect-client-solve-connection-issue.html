Risoluzione di un problema di connessione di Pulse Secure Client

Per facilitare la risoluzione dei problemi di connessione, è possibile utilizzare le seguenti informazioni.

Problema

Descrizione e suggerimenti per la risoluzione

L'URL per una connessione viene visualizzato come Selezionato a caso.

Se l'URL del server viene visualizzato come Selezionato a caso, significa che la connessione è stata definita dall'amministratore di Pulse Secure Client. Invece che con un server di destinazione, la connessione viene configurata con un elenco di server Pulse Secure Client che vengono selezionati in maniera casuale. Viene visualizzato un pulsante Modifica nei dettagli della connessione per ogni connessione che include un elenco di server. L'amministratore di Pulse Secure Client può configurare l'elenco in modo che Pulse Secure Client si connetta sempre al primo server dell'elenco, al server a cui si è effettuato l'accesso più di recente o a un server scelto casualmente dall'elenco.

Pulse Secure Client ha disabilitato le connessioni wireless e viene visualizzato Disattivato - connessione cablata attiva.

L'amministratore di Pulse Secure Client può configurare le connessioni in modo che le connessioni wireless funzionino solo in assenza di una connessione cablata disponibile. Non appena si connette il computer a una connessione cablata, ad esempio connettendolo a un alloggiamento di espansione, Pulse Secure Client interrompe la connessione wireless e utilizza quella cablata.

Dopo l'installazione di Pulse Secure Client e la disinstallazione di Odyssey Access Client (OAC), il sistema non è più in grado di connettersi a una rete wireless.

Pulse Secure Client sostituisce i servizi di accesso forniti da OAC. Tuttavia, OAC gestisce una connettività wireless del computer e Pulse Secure Client si basa sulla connettività fornita da Windows. Dopo la disinstallazione di OAC, è necessario impostare le proprietà di rete wireless di Windows per gestire le reti wireless.

Durante l'accesso, viene visualizzato il seguente messaggio:

Verrà autenticato un server non attendibile. Accettare il certificato per questa connessione?

Pulse Secure Client non è in grado di verificare l'identità del server. Questo errore o qualsiasi altro errore relativo ai certificati indica che Pulse Secure Client non è in grado di garantire l'attendibilità del server a cui si sta eseguendo la connessione. È possibile che il certificato del server sia stato revocato o sia scaduto. Potrebbe essere stato emesso da un'autorità di certificazione non riconosciuta da Pulse Secure Client, ad esempio nel caso in cui l'organizzazione utilizzi un certificato autofirmato. Se l'amministratore di rete ha abilitato tale autorizzazione, è possibile scegliere di procedere e connettersi al server, ma questa opzione è consigliabile solo se l'amministratore di rete ha comunicato di ignorare l'errore.

Dopo l'accesso, viene visualizzato il seguente messaggio:

Accesso limitato alla rete a causa di problemi di conformità.

L'endpoint non soddisfa uno o più requisiti richiesti dai criteri di accesso alla rete. Se le informazioni di correzione di Pulse Secure Client non sono sufficienti per consentire la risoluzione del problema, contattare l'amministratore di rete. Una volta risolto il problema, Pulse Secure Client eseguirà automaticamente la connessione alla rete.

Durante l'accesso, viene visualizzato il seguente messaggio:

Credenziali non valide. Riprovare.

Se l'utente ha selezionato la casella di controllo Salva impostazioni quando ha attivato una connessione Pulse Secure Client, le credenziali fornite vengono utilizzate ogni volta che attiva tale connessione senza che venga richiesta l'immissione delle informazioni di accesso. Tuttavia, Pulse Secure Client non può rilevare quando viene modificata la password di rete, continua a usare le impostazioni salvate e questo può determinare un errore "Credenziali non valide". Per risolvere questo problema, fare clic su File > Connessioni > Dimentica impostazioni salvate.

Alla successiva connessione, verrà richiesto di immettere le credenziali di accesso e sarà possibile specificare le nuove informazioni. Selezionando la casella di controllo Salva impostazioni, verranno salvate le informazioni relative alle nuove credenziali.

L'icona nell'area di notifica e un'icona nel pannello Connessioni cambiano per segnalare lo stato di errore:

Connessione non riuscita

Quando nel pannello Connessioni appare l'icona di connessione non riuscita, fare clic sulla connessione per visualizzarne lo stato. Nella sezione Dettagli è riportato l'errore specifico ed è possibile fare clic su di esso per aprire una finestra contenente la relativa descrizione dettagliata.

Il tentativo di accesso non riesce quando si specifica l'area di autenticazione di destinazione come parte del nome utente. Ad esempio:

nomeutente@areadiautenticazione

La convenzione nomeutente@areadiautenticazione non è supportata.
