Pulse Secure Client Log Files

A Pulse Secure client log file tracks information that can help solve connection issues.

Log files include your Pulse Secure client connections and their properties. If you want to save the configuration, just save the logs. Your current Pulse Secure client configuration is saved in the log package. If you uninstall Pulse Secure client, the program asks you if you want to save your Pulse Secure client configuration so that when you install Pulse Secure client again, your Pulse Secure client connections and other configuration settings are still there.

Logging is a background operation. You do not need to make any changes to your logging environment unless instructed to do so as part of a troubleshooting effort.

In addition to the Pulse Secure client log files, Pulse Secure client writes events to the Windows application event log (Windows Vista and Windows 7 systems only). If you have administrator access to your computer, you can use the Windows Event Viewer to view Pulse Secure client events that have been logged to the event log. Event log information is included when you save the Pulse Secure client log files.

To help in a troubleshooting effort, you might be asked to do the following tasks:

  • Annotate the logs—When you annotate a log file, you insert text that marks the log file at a specific location. For example, to troubleshoot a connection problem, you might be asked to annotate the log file with specific text, attempt the connection that has been failing, and then annotate the log file again. This sequence of events allows a support representative to search the log file for the text you inserted. The text brackets the entries of the log file that track your connection issue.
  • Set the log level—The default logging level is Normal. For a troubleshooting operation, you might be asked to change the logging level to Detailed.
  • Save the logs—The Save As operation gathers all the log files into a single ZIP file and lets you specify where to place the file.
