Troubleshoot a Pulse Secure Client Connection Issue

You can use the following troubleshooting information to help resolve connection issues.

Issue

Description and Resolution Suggestions

The URL for a connection appears as Selected at random.

If the Server URL appears as Selected at random, it means that this connection was defined by the Pulse Secure client administrator. Instead of one target server, this connection is configured with a list of Pulse Secure client servers and Pulse Secure client randomly selects the target server. A Change button appears in the connection details for every connection that includes a list of servers. Your Pulse Secure client administrator can configure the list so that Pulse Secure client always connects to the first server in the list, to the server where you were most recently connected, or to a server selected randomly from the list.

Pulse Secure client has disabled wireless connections and displays Disabled – Wired Connection Active.

The Pulse Secure client administrator can configure Pulse Secure client connections so that wireless connections work only when there is no wired connection available. As soon as you connect your computer to a wired connection, for example, by connecting it to a docking station, Pulse Secure client drops the wireless connection and uses the wired connection.

After installing Pulse Secure client and uninstalling Odyssey Access Client (OAC), your system can no longer connect to any wireless network.

Pulse Secure client replaces the access services provided by OAC. However, OAC manages a computer’s wireless connectivity and Pulse Secure client relies on the connectivity provided by Windows. After uninstalling OAC, you must set Windows wireless networking properties to manage wireless networks.

During login, the following message appears:

You are about to authenticate to an untrusted server. Do you accept the certificate for this connection?

Pulse Secure client cannot verify the identity of the server. This error or any certificate error means that Pulse Secure client cannot ensure that you are connecting to a trusted server. The server certificate might have been revoked or it might have expired. It could have been issued by a certificate authority that is not recognized by Pulse Secure client, such as when your organization uses a self-signed certificate. If your network administrator has enabled this permission, you can choose to proceed and connect to the server but you should do so only if your network administrator has advised you to ignore the certificate error.

After login, the following message appears:

Limited network access due to compliance issue

Your endpoint does not meet one or more of the requirements that your network access policies require. If the Pulse Secure client Remediation information does not provide enough information to enable you to solve the issue, see your network administrator. When you fix the problem, Pulse Secure client will automatically connect you to the network.

During login, the following message appears:

Credentials were invalid. Please try again.

If you selected the Save Settings check box when you activated a Pulse Secure client connection, the credentials you provided are used every time you activate that connection without prompting you to enter login information. However, Pulse Secure client cannot detect when you change your network password — it continues to use the saved settings, and that can result in the “Credentials were invalid” error. To resolve this issue, click File > Connections > Forget Saved Settings.

The next time you connect, you are prompted for your login credentials, and you can specify your new login information. If you select the Save Settings check box, the new credential information will be saved.

The system tray icon and a Connections pane icon change to the failed state:

Connection failed

When you see the connection failed icon in the Connections pane, click the connection to display the connection status. The Details section shows the specific error, which you can click to open a window that shows a detailed description of the error.

Your login attempt fails when you specify the target realm as part of your username. For example:

username@realm

The username@realm convention is not supported.
