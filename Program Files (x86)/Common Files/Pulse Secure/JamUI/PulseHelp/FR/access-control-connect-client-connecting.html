Se connecter à un réseau

Au moins une connexion doit être configurée pour que vous puissiez vous connecter à un réseau. Les invites affichées dépendent de votre environnement d'accès réseau.

Pour utiliser Pulse Secure Client avec un réseau sans fil, vous devrez peut-être configurer au préalable les paramètres réseau sans fil de votre point terminal via Windows ou le logiciel du dispositif sans fil installé sur votre point terminal. L'administrateur réseau peut également définir vos connexions sans fil et listes d'analyse, et les inclure dans votre installation Pulse Secure Client.

Si l'adresse URL du serveur apparaît comme Sélectionnée de manière aléatoire, cela signifie que cette connexion a été définie par l'administrateur Pulse Secure Client. Cette connexion est configurée non pas avec un seul serveur cible, mais avec une liste de serveurs Pulse Secure Client dans laquelle ce dernier sélectionne le serveur cible de manière aléatoire. Un bouton Changer apparaît dans les détails de connexion pour toutes les connexions comportant une liste de serveurs. Cliquez sur le bouton Changer pour sélectionner le serveur cible dans la liste. Votre administrateur Pulse Secure Client peut configurer la liste de sorte que Pulse Secure Client se connecte toujours au premier serveur de la liste, au dernier serveur auquel vous vous êtes connecté ou à un serveur sélectionné de manière aléatoire dans la liste.

Pour utiliser une connexion définie en vue de se connecter à un réseau :

  1. Cliquez sur l'option Connexion en regard de la connexion à établir.

    Ou

    Dans la barre d'état système, cliquez avec le bouton droit de la souris sur l'icône Pulse Secure Client pour afficher le menu contextuel, puis cliquez sur la connexion et sur Connexion.

  2. Répondez aux demandes d'informations telles que nom d'utilisateur et mot de passe, domaine, rôle, etc. Ces demandes dépendent de la manière dont votre administrateur réseau a configuré votre environnement d'accès.

  3. Une boîte de dialogue d'état de la connexion s'affiche. Si la tentative de connexion a réussi, vous pouvez cliquer sur Fermer ou attendre la fermeture automatique de la boîte de dialogue.

    En cas de problème de connexion, une boîte de dialogue de correction affiche la procédure à suivre pour résoudre le problème ou une boîte de dialogue de correction distincte affiche la progression des mesures prises automatiquement par Pulse Secure Client pour résoudre le problème.

Après avoir cliqué sur le bouton Connexion, vous devrez peut-être répondre aux invites suivantes :

  • Certificat : si Pulse Secure Client doit communiquer avec un serveur de certificats et que votre administrateur réseau a configuré plusieurs serveurs, il vous est demandé de choisir un serveur. Un certificat émis par une autorité de certification vérifie que la ressource réseau à laquelle vous vous connectez est valide. Si le certificat provient d'une source certifiée, il est accepté automatiquement et l'invite de certificat ne s'affiche pas. En cas de problème lié au certificat, vous serez peut-être invité à confirmer que vous acceptez le certificat et que vous souhaitez poursuivre la connexion.
  • Domaine : un domaine définit les ressources réseau auxquelles vous pouvez accéder. Si vous avez accès à plusieurs domaines, vous devez en sélectionner un.
  • Données d'identification : vos nom d'utilisateur et mot de passe ou vos nom d'utilisateur et code de jeton établissent votre identité auprès du dispositif d'accès. Un nom d'utilisateur et un mot de passe secondaires ou un nom d'utilisateur et un mot de passe de serveur proxy peuvent également vous être demandés. Votre environnement d'authentification peut vous inviter périodiquement à modifier votre mot de passe ou votre code PIN de jeton.
  • Rôles : un rôle définit les paramètres de session (valeurs d'expiration de la session, nombre de sessions simultanées autorisées, etc.), les paramètres de personnalisation et les fonctions d'accès activées. Votre administrateur réseau peut vous permettre de choisir le rôle souhaité lorsque vous vous connectez.
  • Sessions simultanées : si vous vous connectez au système plusieurs fois, par exemple, si vous utilisez les mêmes données d'identification pour vous connecter à partir de deux ordinateurs différents, il peut vous être demandé de mettre fin à la session en cours avant d'en commencer une autre.
  • Actions de correction : il peut vous être demandé d'effectuer certaines actions pour assurer la conformité de votre point terminal aux règles de sécurité.

La case à cocher Enregistrer les paramètres peut s'afficher dans chaque écran de connexion. (Votre administrateur peut désactiver cette fonction.) Si vous cochez cette case, vous ne serez pas invité à saisir ces informations lors de votre prochaine connexion. Si vous enregistrez les paramètres, vous pouvez utiliser la fonction Effacer les paramètres enregistrés pour être à nouveau invité à saisir les informations de connexion. Notez que l'option de suspension de connexion n'est pas disponible si vous choisissez d'effacer les paramètres enregistrés alors qu'une connexion est active. La fonction Enregistrer les paramètres vous permet d'enregistrer les informations suivantes :

  • Validation du certificat
  • Sélection du certificat
  • Domaine
  • Nom d'utilisateur et mot de passe
  • Nom d'utilisateur/mot de passe du serveur proxy
  • Nom d'utilisateur/mot de passe secondaires
  • Rôle

Les étapes qui suivent vos réponses aux invites de connexion dépendent des stratégies d'accès configurées par l'administrateur réseau et du type de dispositif d'accès au réseau. Le processus de connexion peut inclure les tâches suivantes :

  • Mises à jour de logiciels : Pulse Secure Client peut être automatiquement mis à jour à chaque connexion. Votre système peut recevoir le logiciel Pulse Secure Client mis à jour ou vous pouvez recevoir des modules logiciels supplémentaires prenant en charge des services étendus, par exemple lorsque vous vous connectez à un nouveau type de dispositif d'accès au réseau pour la première fois.
  • Host Checker : Host Checker analyse votre point terminal pour déterminer s'il est conforme aux règles de sécurité de votre organisation. Par exemple, Host Checker peut contrôler la présence de logiciel antivirus et de signatures de virus à jour, l'état du pare-feu personnel, ou encore la présence de logiciels espions et de programmes malveillants. Il peut également effectuer certaines tâches comme désactiver la fonction de saisie semi-automatique du navigateur et vider votre corbeille. Ces paramètres sont définis par l'administrateur réseau. Si Host Checker détecte un problème, il affiche un message décrivant les étapes que vous devez suivre pour résoudre ce problème, ou il lance automatiquement les opérations de correction.
  • Cache Cleaner : Cache Cleaner supprime les fichiers temporaires stockés par Internet Explorer. Cache Cleaner est exécuté en tant qu'option de Host Checker.
