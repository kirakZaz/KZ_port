Présentation de Pulse Secure Client

Pulse Secure Client permet d'établir une connexion authentifiée sécurisée aux ressources réseau protégées au sein des réseaux locaux et des réseaux étendus. Pulse Secure Client s'intègre également avec Pulse Secure Client Collaboration Suite pour les réunions en ligne. Pulse Secure Client simplifie la connectivité réseau en laissant l'administrateur réseau configurer votre environnement d'accès réseau.

Une connexion sécurisée chiffre le trafic réseau de manière à ce que seuls votre point terminal et le serveur avec lequel il communique en comprennent le contenu. L'administrateur réseau définit les paramètres de sécurité utilisés par Pulse Secure Client lorsqu'il établit une connexion réseau.

Une connexion authentifiée signifie que votre point terminal et le serveur se sont mutuellement prouvé leur identité. Le serveur utilise un certificat émis par une autorité de certification approuvée pour s'authentifier, et votre point terminal présente vos données d'identification de connexion au serveur. L'administrateur réseau peut configurer votre environnement pour que l'authentification s'effectue automatiquement sans aucune information à fournir de votre part, bien que généralement, vous soyez invité à fournir vos nom d'utilisateur et mot de passe lorsque vous vous connectez.

Les ressources réseau protégées sont tous les périphériques ou applications du réseau auxquels vous accédez via une connexion authentifiée. Par exemple, pour accéder au système de messagerie de votre organisation, vous vous connectez d'abord à votre réseau. Pulse Secure Client vous permet également de lancer le client Pulse Secure Client Collaboration Suite et de participer à des réunions. L'administrateur réseau répartit les ressources réseau protégées en domaines et vous permet d'y accéder en accordant l'accès à des domaines sélectionnés. Si vous avez accès à plus d'un domaine, vous pouvez sélectionner le domaine souhaité lorsque vous vous connectez au réseau via Pulse Secure Client. L'administrateur réseau peut également définir les propriétés de sécurité nécessaires pour accéder à un domaine en activant le programme Host Checker. Host Checker s'exécute lorsque vous vous connectez au réseau via Pulse Secure Client. Il examine votre point terminal pour s'assurer qu'il dispose du logiciel requis, notamment une protection à jour contre les virus et les logiciels malveillants. Votre administrateur peut également configurer vos connexions Pulse Secure Client pour qu'elles autorisent uniquement les accès aux applications basées sur un réseau spécifique.

L'administrateur réseau peut configurer les stratégies d'accès pour qu'une fois connecté au réseau, vous disposiez d'une connexion portable. Par exemple, si vous connectez votre ordinateur portable au réseau de l'entreprise à partir de votre domicile, il n'est pas nécessaire de vous déconnecter puis de vous reconnecter lorsque vous amenez votre ordinateur portable au bureau. Il vous suffit de fermer votre ordinateur portable chez vous. Lorsque vous arrivez au bureau et que vous le rouvrez, Pulse Secure Client détecte le nouvel environnement réseau. Pulse Secure Client utilise les informations de votre profil de connexion afin de configurer votre connexion pour un accès approprié et vous authentifie dans le nouvel environnement. Si l'administrateur a configuré la fonction de reconnaissance d'emplacement de Pulse Secure Client, vous serez connecté au réseau approprié au démarrage. Par exemple, si votre point terminal est démarré dans un emplacement distant, Pulse Secure Client établit une connexion à un dispositif d'accès de réseau étendu automatiquement.

Pulse Secure Client permet à l'administrateur réseau d'envoyer de nouveaux logiciels et des mises à jour logicielles importantes à votre point terminal lorsque vous vous connectez au réseau.
