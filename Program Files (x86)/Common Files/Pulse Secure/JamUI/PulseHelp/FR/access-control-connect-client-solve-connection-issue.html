Résoudre un problème de connexion Pulse Secure Client

Pour faciliter la résolution des problèmes de connexion, vous pouvez utiliser les informations de dépannage suivantes.

Problème

Description et solutions suggérées

L'adresse URL pour une connexion s'affiche comme Sélectionnée de manière aléatoire.

Si l'adresse URL du serveur apparaît comme Sélectionnée de manière aléatoire, cela signifie que cette connexion a été définie par l'administrateur Pulse Secure Client. Cette connexion est configurée non pas avec un seul serveur cible, mais avec une liste de serveurs Pulse Secure Client, puis Pulse Secure Client sélectionne le serveur cible de manière aléatoire. Un bouton Changer apparaît dans les détails de connexion pour toutes les connexions comportant une liste de serveurs. Votre administrateur Pulse Secure Client peut configurer la liste de sorte que Pulse Secure Client se connecte toujours au premier serveur de la liste, au dernier serveur auquel vous vous êtes connecté ou à un serveur sélectionné de manière aléatoire dans la liste.

Pulse Secure Client a désactivé les connexions sans fil et affiche Désactivé - Connexion filaire active.

L'administrateur Pulse Secure Client peut configurer les connexions Pulse Secure Client pour que les connexions sans fil ne s'activent que lorsqu'aucune connexion filaire n'est disponible. Dès que vous connectez votre ordinateur à une connexion filaire, par exemple à une station d'accueil, Pulse Secure Client abandonne la connexion sans fil au profit de la connexion filaire.

Après l'installation de Pulse Secure Client et la désinstallation d'Odyssey Access Client (OAC), votre système ne peut plus se connecter à aucun réseau sans fil.

Pulse Secure Client remplace les services d'accès fournis par OAC. Néanmoins, OAC gère la connectivité sans fil d'un ordinateur et Pulse Secure Client s'appuie sur la connectivité fournie par Windows. Après avoir désinstallé OAC, vous devez définir des propriétés de mise en réseau sans fil Windows pour gérer les réseaux sans fil.

Lors de la connexion, le message suivant s'affiche :

Vous êtes sur le point de procéder à l'authentification au niveau d'un serveur non certifié. Acceptez-vous le certificat de la connexion ?

Pulse Secure Client ne peut pas vérifier l'identité du serveur. Cette erreur ou toute erreur de certificat signifie que Pulse Secure Client ne peut pas garantir que vous soyez connecté à un serveur approuvé. Il se peut que le certificat de serveur ait été révoqué ou qu'il ait expiré. Il peut avoir été délivré par une autorité de certification non reconnue par Pulse Secure Client, comme lorsque votre organisation utilise un certificat autosigné. Si l'administrateur réseau a activé cette autorisation, vous pouvez choisir de poursuivre et de vous connecter au serveur, mais seulement si l'administrateur réseau vous a conseillé d'ignorer l'erreur de certificat.

Une fois la connexion établie, le message suivant s'affiche :

Accès réseau limité en raison d'un problème de conformité

Votre point terminal ne respecte pas une ou plusieurs des conditions requises par vos stratégies d'accès réseau. Si les informations relatives à la correction Pulse Secure Client ne sont pas suffisantes pour vous permettre de résoudre le problème, contactez l'administrateur réseau. Une fois le problème résolu, Pulse Secure Client vous connecte automatiquement au réseau.

Lors de la connexion, le message suivant s'affiche :

Les données d'identification n'étaient pas valables. Veuillez réessayer.

Si vous avez coché la case Enregistrer les paramètres lors de l'activation d'une connexion Pulse Secure Client, les données d'identification que vous avez fournies sont utilisées chaque fois que vous activez cette connexion, sans que vous soyez invité à saisir à nouveau vos informations de connexion. Pulse Secure Client n'est cependant pas capable de détecter quand vous modifiez votre mot de passe réseau : il continue à utiliser les paramètres enregistrés, ce qui peut engendrer l'erreur « Les données d'identification n'étaient pas valables ». Pour résoudre ce problème, cliquez sur Fichier > Connexions > Effacer les paramètres enregistrés.

Lors de votre prochaine connexion, vous serez invité à saisir vos données d'identification de connexion et vous pourrez spécifier vos nouvelles informations de connexion. Si vous cochez la case Enregistrer les paramètres, les nouvelles données d'identification sont enregistrées.

L'icône de la barre d'état système et une icône du volet Connexions passent à l'état « échec » :

Échec de la connexion

Lorsque l'icône d'échec de connexion apparaît dans le volet Connexions, cliquez sur la connexion pour en afficher l'état. La section Détails indique l'erreur spécifique, sur laquelle vous pouvez cliquer pour ouvrir une fenêtre en affichant une description détaillée.

Votre tentative de connexion échoue lorsque vous spécifiez le domaine cible dans votre nom d'utilisateur. Par exemple :

nom d'utilisateur@domaine

La convention nom d'utilisateur@domaine n'est pas prise en charge.
