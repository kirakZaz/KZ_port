Descripción general de Pulse Secure Client

Pulse Secure Client permite realizar conexiones autenticadas seguras para proteger los recursos de red en redes de área extensas y locales. Pulse Secure Client también se integra con el conjunto Pulse Secure Client Collaboration para reuniones en línea. Pulse Secure Client simplifica la conectividad de red al permitir que el administrador de red configure el entorno de acceso a red.

Una conexión segura codifica el tráfico de la red para que solo el punto final y el servidor con el que se comunique puedan acceder al contenido. El administrador de red define los ajustes de seguridad que emplea Pulse Secure Client cuando establece una conexión de red.

Una conexión autenticada implica que el punto final y el servidor han verificado su identidad mutuamente. El servidor utiliza un certificado de una autoridad de certificado fiable para autenticarse y su punto final presenta sus credenciales de registro en el servidor. El administrador de red puede configurar su entorno de manera que la autenticación ocurra automáticamente sin solicitarle ninguna información, aunque generalmente se le va a pedir su nombre de usuario y contraseña cuando se conecte.

Un recurso de red protegido es cualquier dispositivo o aplicación de red al que acceda mediante una conexión autenticada. Por ejemplo, para acceder al sistema de correo electrónico de la organización, primero debe iniciar sesión en la red. Pulse Secure Client le permite también abrir el cliente de Pulse Collaboration Suite y participar en las reuniones. El administrador de red organiza recursos de red protegidos en territorios y otorga acceso a determinados territorios a fin de autorizar el acceso a dichos recursos. Si tiene acceso a más de un territorio, debe seleccionar el que desea usar al conectarse a la red a través de Pulse Secure Client. El administrador de red también puede definir las propiedades de seguridad necesarias para acceder a un territorio activando el programa Host Checker. Host Checker se ejecuta cuando se conecta a una red mediante Pulse Secure Client. Examina el punto final para asegurarse de que cuenta con el software necesario, como protección contra virus y malware actualizado. El administrador puede también configurar las conexiones de Pulse Secure Client para permitir el acceso solamente a aplicaciones específicas basadas en la red.

El administrador de red puede configurar las directivas de acceso de modo que, después de conectarse a la red, la conexión sea portátil. Por ejemplo, si conecta el equipo portátil a la red de la empresa desde su casa, no es necesario que se desconecte y luego se vuelva a conectar cuando traiga el equipo portátil a la oficina. Simplemente puede cerrar el equipo portátil en casa. Cuando llegue a la oficina y lo abra, Pulse Secure Client percibe el cambio de entorno de red. Pulse Secure Client utiliza la información de su perfil de conexión para configurarla y así conseguir el acceso adecuado y autenticarle en el nuevo entorno. Si el administrador ha configurado la función de consciencia de ubicación de Pulse Secure Client, se conectará con la red adecuada al inicio. Por ejemplo, si el punto final se inicia en una ubicación remota, Pulse Secure Client realiza una conexión de forma automática a un dispositivo de acceso WAN.

Pulse Secure Client permite al administrador de red “cargar” nuevo software y actualizaciones de software importantes en el punto final cuando se conecte a la red.
