Interfaz de Pulse Secure Client

En la interfaz de Pulse Secure Client se indican las conexiones de Pulse Secure Client. Normalmente, el administrador de red define e implementa las conexiones en el cliente de Pulse Secure Client. El administrador puede permitir también crear nuevas conexiones con servidores de Pulse Secure Client. Si se conecta al portal web de un servidor de Pulse Secure Client, puede añadir una conexión a ese servidor en su interfaz de Pulse Secure Client.

Pulse Secure Client muestra también un icono en la bandeja del sistema que muestra el estado de la conexión y que le permite conectarse a redes y reuniones y desconectarse de ellas, así como acceder rápidamente a la interfaz del programa.

Botones de conexión:

Añada una conexión nueva.

Editar una conexión inactiva seleccionada.

Eliminar la conexión inactiva seleccionada.

Iconos de estado de conexión:

Conectado

Conexión

Con conexión, pero con limitaciones

Error al intentar conectarse

Se ha suspendido la conexión

Conectado a la red local, pero sin acceso a Internet disponible. Las redes Wi-Fi públicas suelen implementar un portal cautivo que requiere que el usuario introduzca la información de autenticación o acepte los términos del servicio antes de concederle acceso a la red. Pulse Secure Client detecta la presencia de portales cautivos y no inicia una conexión con un servidor de Pulse Secure Client hasta que no se otorga el acceso a Internet.

Pulse Secure Client es compatible con el estándar FIPS (Federal Information Processing Standard), que define las prácticas de comunicación segura para el gobierno de los EE. UU. Si el administrador de Pulse Secure Client ha habilitado FIPS, aparece FIPS Activado en la parte inferior de la ventana del cliente Pulse Secure.
