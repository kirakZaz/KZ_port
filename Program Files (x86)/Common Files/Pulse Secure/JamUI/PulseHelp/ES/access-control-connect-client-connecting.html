Conectar a una red

Debe tener al menos una conexión configurada para poder conectarse a una red. Los indicadores que ve dependen de su entorno de acceso a la red.

Para usar Pulse Secure Client con una red inalámbrica, es posible que primero necesite configurar los ajustes de red inalámbrica de su punto final con Windows o el software del dispositivo inalámbrico instalado en su punto final. El administrador de red también puede definir sus conexiones inalámbricas y listas de análisis e incluirlas en su instalación de Pulse Secure Client.

Si la URL del servidor muestra Seleccionada al azar, significa que esta conexión fue definida por el administrador de Pulse Secure Client. En lugar de un servidor de destino, esta conexión se configura con una lista de servidores de Pulse Secure Client y Pulse Secure Client selecciona al azar el servidor de destino. El botón Cambiar aparece en los detalles de la conexión para cada conexión que incluye una lista de servidores. Haga clic en el botón Cambiar para seleccionar el servidor de destino en la lista. Su administrador de Pulse Secure Client puede configurar la lista de forma que Pulse Secure Client se conecte siempre al primer servidor de la lista, al servidor al que se había conectado más recientemente o a un servidor seleccionado al azar en la lista.

Para usar una conexión definida para conectarse a una red:

  1. Haga clic en Conectar en la conexión que desea establecer.

    O bien:

    En la bandeja del sistema, haga clic con el botón secundario en el icono de Pulse Secure Client para mostrar el menú emergente. Haga clic en la conexión y, a continuación, en Conectar.

  2. Responda las solicitudes de información como nombre de usuario y contraseña, territorio, función, etc. Las solicitudes que ve dependen de cómo haya configurado su entorno de acceso en administrador de redes.

  3. Aparecerá un cuadro de diálogo con el estado de conexión. Si el intento de conexión se ha realizado correctamente, puede hacer clic en Cerrar o esperar un momento hasta que el cuadro de diálogo se cierre automáticamente.

    Si hay un problema con el intento de conexión, un cuadro de diálogo de corrección va a mostrar información sobre lo que puede hacer para resolver el problema. También puede aparecer un cuadro de diálogo de corrección, que muestra el progreso de las medidas que ha tomado Pulse Secure Client de forma automática para solucionar el problema.

Tras hacer clic en el botón Conectar, puede que necesite responder a las siguientes solicitudes:

  • Certificado: si Pulse Secure Client necesita comunicarse con un servidor de certificados y el administrador de red ha configurado más de un servidor, se le solicitará que elija uno. Un certificado emitido por una autoridad de certificación verifica que el recurso de red al que se conecta es válido. Si el certificado procede de una fuente fiable, se acepta automáticamente y no verá la solicitud de certificado. Si hay un problema con el certificado, es posible que se le pregunte si desea aceptar el certificado y realizar la conexión.
  • Territorio: un territorio define el recurso de red al que se puede acceder. Si tiene acceso a más de un territorio, debe seleccionar uno.
  • Credenciales: su nombre de usuario y contraseña, o nombre de usuario y código token, establecen su identidad para acceder al dispositivo. Puede que también se le pida un nombre de usuario y contraseña secundarios, así como un nombre de usuario y contraseña para un servidor proxy. Su entorno de autenticación puede solicitarle periódicamente que cambie su contraseña y su número PIN de token.
  • Función: una función define los parámetros de la sesión (valores de tiempo de espera, el número de sesiones simultáneas permitido, etc.), ajustes de personalización y características de acceso activadas. El administrador de red puede permitirle escoger la función que desee al conectarse.
  • Sesiones simultáneas: si inicia sesión en el sistema más de una vez, por ejemplo, si utiliza las mismas credenciales para iniciar sesión desde dos equipos diferentes, es posible que el sistema le solicite finalizar la sesión actual antes de comenzar una nueva.
  • Acciones de corrección: puede que se le pida realizar acciones para que el punto final cumpla con las directivas de seguridad.

Puede aparecer una casilla de verificación Configuraciones guardadas en cada pantalla de inicio de sesión. (El administrador puede desactivar esta función). Si activa la casilla de verificación, no se le solicita esa información la próxima vez que inicie sesión. Si guarda los ajustes, puede usar la función Borrar ajustes guardados para volver a que se le solicite información de inicio de sesión. Si guarda los ajustes, puede usar la función Olvidar configuraciones guardadas para que se vuelva a solicitar la información de inicio de sesión. La función Guardar configuración le permite guardar la siguiente información:

  • Aceptación del certificado
  • Selección del certificado
  • Territorio
  • Nombre de usuario y contraseña
  • Nombre de usuario/contraseña de proxy
  • Nombre de usuario/contraseña secundarios
  • Función

Los pasos que se realizan después de responder a todas las solicitudes de conexión dependen de las directivas de acceso que el administrador de red haya configurado y del tipo de dispositivo de acceso a red. El proceso de conexión puede incluir las siguientes tareas:

  • Actualizaciones de software: Pulse Secure Client se puede actualizar automáticamente al conectarse. Su sistema puede recibir el software de Pulse Secure Client actualizado o puede recibir módulos de software adicionales para admitir servicios ampliados, como al conectarse a un nuevo tipo de dispositivo de acceso a red por primera vez.
  • Host Checker: Host Checker examina su punto final para determinar si cumple con las directivas de seguridad de su organización. Por ejemplo, Host Checker puede verificar si tiene un software antivirus y firmas de virus actualizadas, el estado del cortafuegos personal y la presencia de malware y spyware. También puede realizar tareas como desactivar la función Autocompletar del navegador y vaciar la papelera de reciclaje. Estos ajustes los define el administrador de red. Si Host Checker encuentra un problema, puede mostrar un mensaje que describe los pasos que debe seguir para corregir el problema o iniciar operaciones de corrección de forma automática.
  • Cache Cleaner: Cache Cleaner elimina los archivos temporales que almacena Internet Explorer. Cache Cleaner se ejecuta como una opción de Host Checker.
