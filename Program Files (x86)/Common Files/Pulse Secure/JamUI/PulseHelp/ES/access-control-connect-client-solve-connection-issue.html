Solucionar un problema de conexión de Pulse Secure Client

Puede usar la siguiente información de solución de problemas como ayuda para resolver problemas de conexión.

Problema

Descripción y sugerencias de resolución

La dirección URL de la conexión aparece como Seleccionada al azar.

Si la URL del servidor aparece como Seleccionada al azar, significa que esta conexión fue definida por el administrador de Pulse Secure Client. En lugar de un servidor de destino, esta conexión se configura con una lista de servidores de Pulse Secure Client y Pulse Secure Client selecciona al azar el servidor de destino. El botón Cambiar aparece en los detalles de la conexión para cada conexión que incluye una lista de servidores. Su administrador de Pulse Secure Client puede configurar la lista de forma que Pulse Secure Client se conecte siempre al primer servidor de la lista, al servidor al que se había conectado más recientemente o a un servidor seleccionado al azar en la lista.

Pulse Secure Client ha desactivado las conexiones inalámbricas y muestra Desactivado – Conexión por cable activa.

El administrador de Pulse Secure Client puede configurar las conexiones de Pulse Secure Client de manera que las conexiones inalámbricas funcionen solo cuando no haya conexión por cable disponible. Tan pronto como establezca una conexión por cable con el equipo (por ejemplo, conectándolo a una estación de acoplamiento), Pulse Secure Client interrumpe la conexión inalámbrica y utiliza la conexión por cable.

Después de instalar Pulse Secure Client y de desinstalar Odyssey Access Client (OAC), su sistema ya no se podrá conectar a ninguna red inalámbrica.

Pulse Secure Client reemplaza los servicios de acceso que proporciona OAC. No obstante, OAC administra la conectividad inalámbrica de un equipo y Pulse Secure Client depende de la conectividad que ofrece Windows. Tras desinstalar OAC, debe configurar las propiedades de red inalámbrica de Windows para administrar redes inalámbricas.

Durante el inicio de sesión aparece el siguiente mensaje:

Está a punto de autenticar un servidor que no es de confianza. ¿Acepta el certificado para esta conexión?

Pulse Secure Client no puede verificar la identidad del servidor. Este error o cualquier error de certificado significa que Pulse Secure Client no puede garantizar que se esté conectando a un servidor de confianza. Es posible que el certificado del servidor se haya revocado o haya vencido. Puede haberlo emitido una autoridad de certificación que Pulse Secure Client no reconoce, como cuando su organización utiliza un certificado autofirmado. Si su administrador de red ha activado este permiso, puede optar por continuar y conectarse al servidor, pero solo debe hacerlo si su administrador de red le ha indicado que ignore el error de certificado.

Después del inicio de sesión aparece el siguiente mensaje:

Acceso limitado a la red debido a un problema de cumplimiento

Su punto final no cumple uno o más de los requisitos requeridos por sus directivas de acceso de red. Si la información de corrección de Pulse Secure Client no proporciona suficiente información para permitirle resolver el problema, consulte con el administrador de red. Cuando solucione el problema, Pulse Secure Client le conectará automáticamente a la red.

Durante el inicio de sesión aparece el siguiente mensaje:

Las credenciales no eran válidas. Vuelva a intentarlo.

Si ha seleccionado el cuadro de diálogo Configuraciones guardadas al activar la conexión de Pulse Secure Client, las credenciales que proporcionó va a usarse cada vez que active la conexión sin que se le solicite que introduzca la información de inicio de sesión. Sin embargo, Pulse Secure Client no puede detectar cuándo cambia la contraseña de red, sino que continúa usando los ajustes guardados, lo que puede provocar el error “Las credenciales no eran válidas”. Para solucionar este problema, haga clic en Archivo > Conexiones > Olvidar configuraciones guardadas.

La próxima vez que se conecte, se le solicitarán las credenciales de inicio de sesión y podrá especificar su nueva información de inicio de sesión. Si selecciona el cuadro de diálogo Configuraciones guardadas, se guardará la nueva información de credenciales.

El icono de la bandeja del sistema y el icono del Panel de conexiones cambian al estado de error:

Error en la conexión

Cuando vea el icono de error en la conexión en el Panel de conexiones, haga clic en la conexión para mostrar su estado. La sección Detalles muestra el error específico, en el que puede hacer clic para abrir una ventana que muestra una descripción detallada del error.

Su intento de inicio de sesión falla cuando especifica el territorio de destino como parte de su nombre de usuario. Por ejemplo:

nombredeusuario@territorio

No se admite la convención nombredeusuario@territorio.
