認証方法

Pulse 管理者は、以下の認証方法を使用するように Pulse 接続を設定できます。

  • Time-based One-Time Password（TOTP） — Time-based One-Time Password（TOTP）は、共有秘密鍵と現在の時刻を使用してパスワード（トークン）を計算するアルゴリズムです。このパスワードは 1 回だけ使用することができ、その有効期限は短期間で切れます。Pulse Secure デスクトップ クライアントと Pulse Connect Secure（PCS）8.2R5 ゲートウェイでは、Google の TOTP 認証サーバー（Google 認証システム）がサポートされています。

    Google 認証システムは、セカンダリ認証サーバーとしてのみ設定できます。

    エンド ユーザーは、この認証タイプを使用する前に、TOTP サーバーに登録する必要があります。通常、登録時には、Web ブラウザーを使用して TOTP 認証サーバー（PCS ゲートウェアなど）に接続し、プライマリ認証を渡して TOTP 登録キー、QR コード、およびバックアップコードにアクセスします。ユーザーがモバイル デバイスで Google 認証システム アプリを使用して QR コードをスキャンするか登録キーを手動で入力すると、アプリによってそのユーザー向けに 6 桁のトークンが 30 秒ごとに生成されます。このトークンは、Pulse Desktop クライアントでの TOTP 認証時に入力できます。

  • トークン認証 — RSA SecurID トークン認証は、2 要素認証とも呼ばれ、漏洩が非常に難しい認証情報を生成する方法です。トークン コードは、クライアントとサーバーの両方で同時に生成されます。ユーザー名およびユーザーが作成した PIN（個別識別番号）に現在のトークン コードを組み合わせることで、ユーザーはサーバーへのログインを許可されます。PIN とトークン コードでパスコードが構成されます。トークン コードは短期間で期限切れになり、新しく生成されたトークン コードと置き換えられます。はじめてシステムを使用するとき、PIN を作成するよう Pulse から要求があります。また、管理者が PIN を提供している場合は、それを変更するよう要求されます。管理者が PIN を定期的に変更するよう要求している場合、Pulse から新しい PIN を要求されます。通常、PIN を変更するときには、現在のトークン コードを提供しなければなりません。場合によっては、パスコードを入力した後、ネットワークにアクセスする前に、次のトークン コードを入力するよう要求されます。トークン デバイスに新しいトークン コードが表示されるまで待ち、その数字を入力して先へ進みます。

    トークンを生成するデバイスまたはソフトウェアは、オーセンティケーターと呼ばれます。Pulse は、SecurID のハードウェア オーセンティケーターおよびソフトウェア オーセンティケーターの全領域をサポートしています。ハードウェア（物理）オーセンティケーターには、60 秒ごとに新しいトークン コードが表示されます。ログイン時には、トークンを生成するカードまたはキーフォブが必要です。Pulse のログイン ウィンドウでは、ユーザー名、PIN、および現在のトークン コードを入力します。USB オーセンティケーターは、Windows ユーザー名/パスワード認証情報とデジタル証明書を保管できます。デバイスがご利用のコンピューターに接続されると、トークン コードの自動入力が有効になります。PIN を提供すると、Pulse Secure クライアントはデバイスを介して直接、自動的にトークン コードにアクセスします。

    ソフトウェア オーセンティケーターには、スマートフォン アプリケーション、デスクトップ PC およびノート PC 用ソフトウェア、ブラウザー ベースのアプリケーションなどがあります。ソフトウェア オーセンティケーターを使用した Pulse ログイン手順は、セキュリティ管理者のシステム設定により異なります。通常、エンドポイントに SecurID ソフトウェアをインストールしている場合、PIN を入力すると、Pulse は、ソフトウェアと通信して、トークン コードを取得します。

  • スマート カード認証 — スマート カードは、ユーザー認証情報が組み込まれたプラスチックカードです。ご利用の環境が複数のデジタル証明書で設定されている場合、1 つを選択するよう Pulse から要求されます。通常、エンドポイント上でスマート カード リーダーを設定している場合、PIN を入力すると、Pulse は、スマート カード ソフトウェアと通信して、トークン コードを取得します。

  • マシン認証 — マシン認証では、エンドポイントの認証にマシンの認証情報を使用します。認証情報は、マシンがドメインに参加すると Active Directory によって生成されます。エンドポイントは、Windows ドメインのメンバーであることが必要です。管理者が認証プロセスをどう設定したかによって、Windows の認証情報を入力するよう要求される場合があります。

  • 資格情報プロバイダ — Pulse 認証情報プロバイダ統合により、Windows ドメインにログインする際に必要なネットワークに接続できます。例えば、Windows のドメイン コントローラーがファイアウォールの内側にある場合、Pulse は、ドメインにログインする前に、資格情報プロバイダ ログインを使用して Pulse サーバーに接続します。Pulse では、Microsoft の資格情報プロバイダとの統合により、パスワードベースのログインとスマート カードによるログインを実現しています。資格情報プロバイダのインターフェイスは、Windows Vista、Windows 7、または Windows 8 のログイン画面では、タイルとして表示されます。Pulse 接続は、ログイン プロセスで Pulse プロンプトが表示されるように設定できます。例えば、レルムやロールを選択するプロンプトを表示できます。資格情報プロバイダ ログイン中、Pulse Secure クライアントのアップグレード通知と動作は無効になり、接続が確立されるまで延期されます。
