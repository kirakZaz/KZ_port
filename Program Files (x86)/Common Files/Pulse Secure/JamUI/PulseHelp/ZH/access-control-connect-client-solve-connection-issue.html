對 Pulse Secure 用戶端連線問題進行疑難排解

您可以使用下列疑難排解資訊來協助解決連線問題。

問題

說明與解決方法建議

某個連線的 URL 顯示為 [隨機選取]。

如果 [伺服器 URL] 顯示為隨機選取，表示此連線是 Pulse Secure 用戶端管理員定義的。此連線並未設定一個目標伺服器，而是設定列成清單的多個 Pulse Secure 用戶端伺服器，並由 Pulse Secure 用戶端隨機選取目標伺服器。每個包含多個伺服器的連線都會在連線詳細資訊中顯示 [變更] 按鈕。您的 Pulse Secure 用戶端管理員能設定清單，使 Pulse Secure 用戶端一律連線至清單中第一個伺服器、您最近連線的伺服器，或是從清單中隨機選取的伺服器。

Pulse Secure 用戶端已停用無線連線，並顯示已停用 - 有線連線作用中。

Pulse Secure 用戶端管理員可以設定 Pulse Secure 用戶端連線，僅在沒有可用的有線連線時才使用無線連線。舉例來說，若將電腦連線到銜接站，只要電腦連線至有線網路，Pulse Secure 用戶端便會中斷無線連線，並改為使用有線連線。

安裝 Pulse Secure 用戶端與解除安裝 Odyssey Access Client (OAC) 後，您的系統將無法再連線至任何無線網路。

Pulse Secure 用戶端會取代由 OAC 提供的存取服務。但是，OAC 會管理電腦的無線連線，且 Pulse Secure 用戶端需仰賴 Windows 提供的連線。解除安裝 OAC 後，您必須設定 Windows 無線網路屬性才能管理無線網路。

在登入期間，會顯示下列訊息：

您正要驗證不受信任的伺服器。您接受此連線的認證嗎？

Pulse Secure 用戶端無法確認伺服器的身份。此錯誤或任何認證錯誤意味著 Pulse Secure 用戶端無法確保您所連線的是受信任的伺服器。伺服器認證可能已被撤銷，或者可能已過期。它可能是由未經 Pulse Secure 用戶端認可的認證授權單位發行，例如當您的組織使用自助簽名認證時。如果網路管理員已啟用此權限，則您可以選擇繼續並連線至伺服器，但只有在網路管理員建議您忽略認證錯誤的情況下，您才可以執行此操作。

登入後，會顯示下列訊息：

由於遵從性問題而受限的網路存取

您的端點不符合網路存取政策所需的一或多個要求。如果 Pulse Secure 用戶端 [矯正] 資訊未提供足夠的資訊讓您解決問題，請向網路管理員詢問。在修正問題時，Pulse Secure 用戶端會自動將您連線至網路。

在登入期間，會顯示下列訊息：

驗證資料無效。請重試。

如果您在啟動 Pulse Secure 用戶端連線時選取了 [儲存設定] 核取方塊，則每次您啟動該連線時，系統都會使用您提供的驗證資料，而不會提示您輸入登入資訊。不過，Pulse Secure 用戶端無法在您變更網路密碼時進行偵測 - 它會繼續使用已儲存的設定，因而可能會導致發生「驗證資料無效」錯誤。若要解決此問題，請按一下檔案 > 連線 > 忘記儲存的設定。

下次連線時，系統會提示您輸入登入驗證資料，然後您就可以指定新的登入資訊。如果您選取 [儲存設定] 核取方塊，則系統會儲存新的驗證資料資訊。

系統匣圖示與 [連線] 窗格圖示會變更為失敗狀態：

連線失敗

當您在 [連線] 窗格中看到連線失敗圖示時，請按一下該連線以顯示連線狀態。[詳細資訊] 區段會顯示明確的錯誤，您可以按一下來開啟顯示錯誤詳細說明的視窗。

當您將目標領域指定為使用者名稱的一部分時，登入嘗試會失敗。例如：

username@realm

不支援 username@realm 慣例。
