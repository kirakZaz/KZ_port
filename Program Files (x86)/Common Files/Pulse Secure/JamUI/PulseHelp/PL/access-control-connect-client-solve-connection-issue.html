Rozwiązywanie problemów z połączeniem aplikacji Pulse Secure

Poniższe informacje ułatwiają rozwiązywanie problemów z połączeniami.

Problem

Opis i sugestie rozwiązania

Jako adres URL połączenia jest wyświetlana informacja Selected at random (Wybrany losowo).

Jeśli jako adres URL serwera jest wyświetlana informacja Selected at random (Wybrany losowo), oznacza to, że połączenie to zostało określone przez administratora aplikacji Pulse Secure. Zamiast jednego serwera docelowego połączenie to skonfigurowano przy użyciu listy serwerów aplikacji Pulse Secure, a serwer docelowy został z niej wybrany losowo przez aplikację Pulse Secure. W szczegółach wszystkich połączeń zawierających listę serwerów pojawi się przycisk Change (Zmień). Administrator aplikacji Pulse Secure może tak skonfigurować listę, aby aplikacja Pulse Secure zawsze nawiązywała połączenie z pierwszym serwerem na liście, z serwerem, z którym najczęściej nawiązuje się połączenia, lub losowo wybranym na liście.

W aplikacji Pulse Secure zostały wyłączone połączenia bezprzewodowe i pojawił się komunikat Disabled – Wired Connection Active (Wyłączono — Aktywne połączenie przewodowe).

Administrator aplikacji Pulse Secure może skonfigurować połączenia bezprzewodowe aplikacji Pulse Secure, tak aby działały tylko w przypadku, gdy nie ma dostępnych żadnych połączeń przewodowych. Od razu po podłączeniu komputera za pomocą połączenia przewodowego (np. do stacji dokującej) aplikacja Pulse Secure kończy połączenie bezprzewodowe i rozpoczyna korzystanie z połączenia przewodowego.

Po zainstalowaniu programu Pulse Secure i odinstalowaniu klienta Odyssey Access Client (OAC) system nie może nawiązać połączenia z żadną siecią bezprzewodową.

Aplikacja Pulse Secure zastępuje usługi dostępowe dostarczane przez klienta OAC. Klient OAC zarządza jednak łącznością bezprzewodową komputera, a aplikacja Pulse Secure jest uzależniona od połączeń obsługiwanych przez system Windows. Po odinstalowaniu klienta OAC we właściwościach sieci bezprzewodowych systemu Windows należy ustawić opcję zarządzania sieciami bezprzewodowymi.

Podczas logowania wyświetlany jest następujący komunikat:

You are about to authenticate to an untrusted server (Nastąpi uwierzytelnienie na niezaufanym serwerze). Do you accept the certificate for this connection (Czy akceptujesz certyfikat tego połączenia)?

Pulse Secure client cannot verify the identity of the server (Aplikacja Pulse Secure nie może zweryfikować tożsamości serwera). Pojawienie się tego błędu lub jakiegokolwiek błędu certyfikatu oznacza, że aplikacja Pulse Secure nie może sprawdzić, czy łączy się z zaufanym serwerem. Być może certyfikat serwera został wycofany lub upłynęła jego ważność. Być może został on wystawiony przez urząd certyfikacji nierozpoznawany przez aplikację Pulse Secure (na przykład w przypadku wykorzystania w organizacji certyfikatu z podpisem własnym). Jeśli administrator sieci przyznał takie uprawnienia, możesz wybrać opcję kontynuowania i połączenia się z serwerem, ale tylko wtedy, gdy administrator sieci zezwoli na zignorowanie błędu certyfikatu.

Po zalogowaniu wyświetlany jest następujący komunikat:

Limited network access due to compliance issue (Ograniczony dostęp do sieci z powodu problemu ze zgodnością)

Your endpoint does not meet one or more of the requirements that your network access policies require (Punkt końcowy nie spełnia co najmniej jednego wymogu określonego w zasadach dostępu do sieci). Jeśli informacje w obszarze Remediation (Naprawa) w aplikacji Pulse Secure nie są wystarczające do rozwiązania problemu, skontaktuj się z administratorem sieci. Po rozwiązaniu problemu aplikacja Pulse Secure automatycznie nawiąże połączenie z siecią.

Podczas logowania wyświetlany jest następujący komunikat:

Credentials were invalid (Nieprawidłowe poświadczenia). Please try again (Spróbuj ponownie).

Jeśli podczas uaktywniania połączenia aplikacji Pulse Secure zaznaczono pole wyboru Save Settings (Zapisz ustawienia), podane poświadczenia są używane za każdym razem do aktywacji połączenia bez wyświetlania monitu o wprowadzenie danych logowania. Aplikacja Pulse Secure nie ma jednak możliwości wykrycia zmiany hasła dostępu do sieci — w dalszym ciągu używa zapisanych ustawień, co może spowodować wystąpienie błędu „Credentials were invalid” (Poświadczenia były nieprawidłowe). Aby rozwiązać ten problem, kliknij kolejno opcje File > Connections > Forget Saved Settings (Plik > Połączenia > Pomiń zapisane ustawienia).

Podczas następnej próby nawiązania połączenia zostanie wyświetlony monit o wprowadzenie poświadczeń logowania, gdzie można będzie określić nowe dane logowania. Jeśli zaznaczysz pole wyboru Save Settings (Zapisz ustawienia), nastąpi zapisanie nowych poświadczeń.

Ikony w zasobniku systemowym i w panelu Connections (Połączenia) przechodzą w stan niepowodzenia:

Connection failed (Połączenie się nie powiodło)

Jeśli w panelu Connections (Połączenia) jest wyświetlana ikona niepowodzenia, kliknij dane połączenie, aby poznać jego stan. W obszarze Details (Szczegóły) zostanie wyświetlony określony błąd, po kliknięciu którego zostanie otwarte okno ze szczegółowym opisem błędu.

Próba logowania nie powiedzie się, jeśli obszar docelowy zostanie podany jako część nazwy użytkownika. Przykład:

nazwa_użytkownika@obszar

Konwencja nazwa_użytkownika@obszar nie jest obsługiwana.
