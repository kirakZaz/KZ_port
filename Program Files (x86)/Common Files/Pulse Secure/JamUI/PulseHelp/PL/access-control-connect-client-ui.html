Interfejs aplikacji Pulse Secure

W oknie aplikacji Pulse Secure znajduje się lista połączeń tej aplikacji. Definiowanie i wdrażanie połączeń w kliencie Pulse należy zwykle do zadań administratora sieci. Administrator może również zezwolić użytkownikowi na tworzenie nowych połączeń z serwerami aplikacji Pulse Secure. Połączenie nawiązane z portalem internetowym serwera aplikacji Pulse Secure może zostać dodane w oknie aplikacji Pulse Secure.

Aplikacja Pulse Secure zawiera również ikonę zasobnika systemowego informującą o stanie połączenia i pozwalającą na nawiązywanie oraz kończenie połączeń z dostępnymi sieciami i spotkaniami, a także zapewniającą szybki dostęp do interfejsu programu.

Przyciski dotyczące połączeń:

Add a new connection (Dodaj nowe połączenie).

Edit a selected inactive connection (Edytuj wybrane nieaktywne połączenie).

Delete the selected inactive connection (Usuń wybrane nieaktywne połączenie).

Ikony stanu połączenia:

Połączono

Łączenie

Połączono z ograniczeniami

Próba nawiązania połączenia się nie powiodła

Połączenie zostało wstrzymane

Połączono z siecią lokalną bez dostępu do Internetu. W miejscach z publiczną siecią Wi-Fi działa często portal uwierzytelniania, który przed przyznaniem użytkownikowi dostępu do sieci wymaga od niego wprowadzenia informacji uwierzytelniających lub akceptacji warunków usługi. Aplikacja Pulse Secure wykrywa obecność portali uwierzytelniania i nie zainicjuje połączenia z serwerem Pulse Secure aż do uzyskania dostępu do Internetu.

Aplikacja Pulse Secure obsługuje standard FIPS (Federal Information Processing Standard), który określa zasady bezpiecznej komunikacji dotyczące rządu Stanów Zjednoczonych. Jeżeli administrator aplikacji Pulse Secure włączył funkcję FIPS, na dole ekranu okna aplikacji Pulse Secure wyświetla się napis FIPS On (FIPS Wł.).
