解决 Pulse Secure 客户端连接问题

您可以使用以下故障排除信息来帮助解决连接问题。

问题

说明和解决方案建议

连接的 URL 显示为“随机选择”。

如果服务器 URL 显示为随机选择，则表示此连接已由 Pulse Secure 客户端管理员定义。此连接配置了一系列 Pulse Secure 客户端服务器（而非一台目标服务器），Pulse Secure 客户端将随机选择目标服务器。包含服务器列表的所有连接都将在连接详细信息中显示“更改”按钮。Pulse Secure 客户端管理员可配置该列表，以便 Pulse Secure 客户端始终连接到列表中的第一台服务器、您最近连接的服务器，或者连接到从列表中随机选择的服务器。

Pulse Secure 客户端已禁用无线连接并显示已禁用 – 有线连接处于活动状态。

Pulse Secure 客户端管理员可配置 Pulse Secure 客户端连接，使无线连接仅在没有有线连接时发挥作用。只要将计算机连接到有线连接，例如将其连接到扩展坞，Pulse Secure 客户端就会断开无线连接，然后使用有线连接。

安装 Pulse Secure 客户端并卸载 Odyssey Access Client (OAC) 后，系统将无法再连接任何无线网络。

Pulse Secure 客户端会取代由 OAC 提供的访问服务。但是，OAC 管理计算机的无线连接，且 Pulse Secure 客户端依赖 Windows 提供的连接。卸载 OAC 后，必须设置 Windows 无线网络属性，才能管理无线网络。

在登录期间，出现以下消息：

您将认证到不可信的服务器。是否接受此连接的证书？

Pulse Secure 客户端无法验证服务器的标识。此错误或任何证书错误意味着 Pulse Secure 客户端无法确保您将连接到受信任的服务器。服务器证书可能已撤消或已过期。该服务器证书可能由未被 Pulse Secure 客户端认可的证书授权机构发布，例如，当组织使用自签证书时。如果网络管理员已启用此权限，则可以选择继续操作并连接到服务器，但只有在网络管理员建议您忽略证书错误后才能这样操作。

登录后，将出现以下消息：

因为合规性问题导致网络访问受限

端点未满足网络访问策略需要的一个或多个要求。如果“Pulse Secure 客户端修复”信息未提供解决问题所需的足够信息，请联系网络管理员。当您修复问题时，Pulse Secure 客户端会自动连接到网络。

在登录期间，出现以下消息：

凭证无效。请重新尝试。

如果在激活某个 Pulse Secure 客户端连接时选中了“保存设置”复选框，则在每次激活该连接时都会使用已提供的凭证，而不会提示您输入登录信息。不过，Pulse Secure 客户端无法检测出您何时更改了网络密码 — 它会继续使用保存的设置，这会导致“凭证无效”错误。要解决此问题，请单击文件 > 连接 > 忘记保存的设置。

下次连接时，系统将提示您输入登录凭证，您可以指定新的登录信息。如果选中“保存设置”复选框，则会保存新的凭证信息。

系统托盘图标和“连接”窗格图标变为故障状态：

连接失败

“连接”窗格中显示连接失败图标时，可单击该连接以显示连接状态。详细信息部分显示了具体错误，可以单击该错误来打开显示错误详细说明的窗口。

将目标领域指定为用户名的一部分时，登录尝试将失败。例如：

username@realm

不支持 username@realm 约定。
