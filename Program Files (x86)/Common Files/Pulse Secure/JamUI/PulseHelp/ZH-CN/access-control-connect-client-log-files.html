Pulse Secure 客户端日志文件

Pulse Secure 客户端日志文件所跟踪的信息有助于解决连接问题。

日志文件包括 Pulse Secure 客户端连接及其属性。如果要保存配置，只需保存日志。当前 Pulse Secure 客户端配置保存在日志包中。如果卸载 Pulse Secure 客户端，程序会询问您是否保存 Pulse Secure 客户端配置，以便在重新安装 Pulse Secure 客户端时，Pulse Secure 客户端连接和其他配置设置仍然存在。

记录操作在后台进行。您无需对记录环境进行任何更改，除非故障排除工作指示您这样做。

除了 Pulse Secure 客户端日志文件，Pulse Secure 客户端还将事件写入 Windows 应用程序事件日志（仅限 Windows Vista 和 Windows 7 系统）。如果您对计算机具有管理员访问权限，则可使用 Windows 事件查看器来查看已记录至事件日志的 Pulse Secure 客户端事件。保存 Pulse Secure 客户端日志文件时，包含事件日志信息。

为协助故障排除工作，支持代表可能会要求您执行以下任务：

  • 注释日志—注释日志文件时，在特定位置插入用于标记日志文件的文本。例如，要解决连接问题，支持代表可能会要求您使用特定文本来注释日志文件，再尝试失败的连接，然后再次注释日志文件。这样的事件顺序可以让支持代表在日志文件中搜索您插入的文本。该文本会将日志文件中用于跟踪连接问题的条目括起。
  • 设置日志级别—默认的记录级别为“常规”。为进行故障排除操作，支持代表可能会要求您将记录级别更改为“详细”。
  • 保存日志—“另存为”操作可以将所有日志文件收集到单个 .zip 文件中，并允许您指定该文件的保存位置。
